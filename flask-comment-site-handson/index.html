STEP 0你将做出什么

成品是一个最小但五脏俱全的评论站:

  • 任何人能看到所有评论(首页)。
  • 能注册账号、登录、退出。
  • 登录后能发表评论,评论带用户名和时间。
  • 密码加盐哈希存储,查询用参数化防注入,模板自动转义防 XSS——主教程讲的三条安全底线全落地。
这一篇用"服务端渲染",和主教程的 fetch 风格互补

主教程 4.3 演示的是"前端用 fetch 调 JSON 接口"的前后端分离风格。这一篇用更适合新手跑通的服务端渲染(SSR):服务器直接把拼好数据的 HTML 整页返回,表单用 POST 提交。两种都是主流,先用 SSR 把整条链路打通,理解后再回头玩 API 风格。

两种风格的区别(一句话)

服务端渲染像"餐厅直接上做好的菜"(服务器把完整页面端给你);前后端分离像"你拿到食材自己摆盘"(服务器只给数据 JSON,前端 JS 负责渲染)。本篇走前者。

STEP 3后端 app.py 逐行

这是项目的大脑。我把它拆成 6 块,每块先讲"这几行干嘛",再给代码。最终把这 6 块按顺序拼进同一个 app.py 文件。

3.1 导入与初始化

引入要用的工具,创建 Flask 应用,设一个用于加密 session 的密钥,指定数据库文件名。

app.py（第 1 块） 
import sqlite3
from datetime import datetime
from flask import Flask, g, request, redirect, url_for, session, render_template, flash
from werkzeug.security import generate_password_hash, check_password_hash

app = Flask(__name__)
# session(登录态)是加密存在用户浏览器 Cookie 里的,需要一把密钥。
# 上线时务必换成一长串随机值,别用下面这句明文!
app.secret_key = "dev-only-change-me-to-random"
DATABASE = "comments.db"
这几行对应主教程哪里

session = 主教程 2.2 的登录态机制;generate_password_hash = 4.8 的"密码加盐哈希";sqlite3 = 4.4 的数据库(SQLite 是个零配置、存成单个文件的 SQL 数据库,练手最省事)。

3.2 数据库连接助手

每次请求按需打开数据库连接,请求结束自动关闭。row_factory 让查询结果能用 列名 访问(像字典),比记下标顺手。

app.py（第 2 块） 
def get_db():
    # g 是 Flask 的"本次请求临时储物格",一次请求里复用同一个连接
    if "db" not in g:
        g.db = sqlite3.connect(DATABASE)
        g.db.row_factory = sqlite3.Row   # 结果可按列名取,如 row["username"]
    return g.db

@app.teardown_appcontext
def close_db(exc):
    # 每次请求结束自动调用,关掉连接
    db = g.pop("db", None)
    if db is not None:
        db.close()

3.3 建表(第一次运行时自动建)

定义两张表:users(用户)和 comments(评论)。IF NOT EXISTS 保证重复运行也不报错。

app.py（第 3 块） 
def init_db():
    db = sqlite3.connect(DATABASE)
    db.executescript("""
    CREATE TABLE IF NOT EXISTS users (
        id            INTEGER PRIMARY KEY AUTOINCREMENT,
        username      TEXT UNIQUE NOT NULL,
        password_hash TEXT NOT NULL
    );
    CREATE TABLE IF NOT EXISTS comments (
        id         INTEGER PRIMARY KEY AUTOINCREMENT,
        user_id    INTEGER NOT NULL,
        text       TEXT NOT NULL,
        created_at TEXT NOT NULL,
        FOREIGN KEY (user_id) REFERENCES users(id)
    );
    """)
    db.commit()
    db.close()

# 模块一加载就确保表存在(本地运行、gunicorn 上线都会执行到)
init_db()
术语:UNIQUE / FOREIGN KEY

username TEXT UNIQUE = 用户名不能重复(数据库层面兜底);FOREIGN KEY (user_id) = 声明"这条评论的 user_id 对应 users 表的某个 id",把两张表关联起来(主教程 4.4 说的"关系型")。

3.4 首页:列出所有评论

用一条 SQL 把评论和发评论的用户名连表查出来,按最新在前,交给模板渲染。

app.py（第 4 块） 
@app.route("/")
def index():
    db = get_db()
    comments = db.execute(
        "SELECT c.text, c.created_at, u.username "
        "FROM comments c JOIN users u ON c.user_id = u.id "
        "ORDER BY c.id DESC"          # 最新的排最前
    ).fetchall()
    return render_template("index.html", comments=comments)
看懂这条 SQL

JOIN ... ON c.user_id = u.id 把评论表和用户表"拼接":每条评论顺带把它作者的用户名取出来。这样首页就能显示"某某:评论内容",而 comments 表里只存了 user_id。

3.5 注册 / 登录 / 退出

三个路由。注册时把密码哈希后入库;登录时核对哈希,通过就把用户写进 session。注意所有 SQL 都用 ? 占位的参数化查询。

app.py（第 5 块） 
@app.route("/register", methods=["GET", "POST"])
def register():
    if request.method == "POST":          # 提交了注册表单
        username = request.form["username"].strip()
        password = request.form["password"]
        if not username or not password:
            flash("用户名和密码不能为空")
            return redirect(url_for("register"))
        db = get_db()
        try:
            db.execute(
                "INSERT INTO users (username, password_hash) VALUES (?, ?)",
                (username, generate_password_hash(password)),   # 存哈希,不存明文!
            )
            db.commit()
        except sqlite3.IntegrityError:        # 触发 UNIQUE 约束 = 用户名已存在
            flash("用户名已被占用")
            return redirect(url_for("register"))
        return redirect(url_for("login"))     # 注册成功,去登录
    return render_template("register.html")  # GET:显示注册页


@app.route("/login", methods=["GET", "POST"])
def login():
    if request.method == "POST":
        username = request.form["username"].strip()
        password = request.form["password"]
        db = get_db()
        user = db.execute(
            "SELECT * FROM users WHERE username = ?", (username,)
        ).fetchone()
        # 用户不存在,或密码哈希对不上 —— 都给同样的提示(别泄露是哪错了)
        if user is None or not check_password_hash(user["password_hash"], password):
            flash("用户名或密码错误")
            return redirect(url_for("login"))
        session["user_id"] = user["id"]       # 登录成功,记住是谁
        session["username"] = user["username"]
        return redirect(url_for("index"))
    return render_template("login.html")


@app.route("/logout")
def logout():
    session.clear()                           # 清掉登录态
    return redirect(url_for("index"))
为什么登录失败不说"是用户名错还是密码错"

统一回"用户名或密码错误",能避免攻击者靠提示语枚举出哪些用户名存在。这种小细节就是安全意识的体现。

3.6 发表评论 + 启动

发评论前先检查有没有登录(session 里有没有 user_id);没登录就打回登录页。最后是本地启动入口。

app.py（第 6 块） 
@app.route("/comment", methods=["POST"])
def comment():
    if "user_id" not in session:        # 没登录,不许发
        return redirect(url_for("login"))
    text = request.form["text"].strip()
    if text:                              # 空评论忽略
        db = get_db()
        db.execute(
            "INSERT INTO comments (user_id, text, created_at) VALUES (?, ?, ?)",
            (session["user_id"], text, datetime.now().strftime("%Y-%m-%d %H:%M")),
        )
        db.commit()
    return redirect(url_for("index"))     # 发完回首页,看到自己的评论


if __name__ == "__main__":
    # 本地开发用:debug=True 改代码自动重载、报错有详情(上线要关掉)
    app.run(debug=True, port=8000)
小结:app.py 全部 6 块按顺序拼好,就是完整后端

顺序:导入初始化 → get_db/close_db → init_db → index → 注册/登录/退出 → 评论/启动。一共约 80 行,这就是一个动态网站后端的全部。

STEP 4三个前端模板

Flask 默认从 templates/ 目录找模板,用 Jinja2 语法:{{ 变量 }} 输出值,{% 逻辑 %} 写条件/循环。重点:{{ }} 会自动转义 HTML,所以用户评论里就算写了 <script> 也只会当文字显示——这就是免费的防 XSS。

4.1 首页 templates/index.html

templates/index.html 
<!DOCTYPE html>
<html lang="zh">
<head>
  <meta charset="UTF-8">
  <title>大家的留言板</title>
  <style>
    body{font-family:serif;max-width:680px;margin:40px auto;padding:0 16px;line-height:1.7;}
    .c{border-bottom:1px solid #eee;padding:10px 0;}
    .who{color:#0b5750;font-weight:bold;} .when{color:#999;font-size:13px;}
    textarea{width:100%;height:64px;} .err{color:#b23b34;}
  </style>
</head>
<body>
  <h1>大家的留言板</h1>

  <!-- 登录与否,显示不同内容 -->
  {% if session.username %}
    <p>你好,<b>{{ session.username }}</b>
       | <a href="{{ url_for('logout') }}">退出</a></p>
    <form method="post" action="{{ url_for('comment') }}">
      <textarea name="text" placeholder="说点什么..." required></textarea>
      <button type="submit">发表</button>
    </form>
  {% else %}
    <p><a href="{{ url_for('login') }}">登录</a>
       或 <a href="{{ url_for('register') }}">注册</a> 后发表评论</p>
  {% endif %}

  <!-- 显示提示信息(如"用户名已占用") -->
  {% for m in get_flashed_messages() %}
    <p class="err">{{ m }}</p>
  {% endfor %}

  <!-- 评论列表:循环渲染 -->
  {% for c in comments %}
    <div class="c">
      <span class="who">{{ c.username }}</span>
      <span class="when">{{ c.created_at }}</span><br>
      {{ c.text }}
    </div>
  {% else %}
    <p>还没有评论,来抢沙发。</p>
  {% endfor %}
</body></html>
Jinja2 的 for...else

{% for %}...{% else %}...{% endfor %} 里的 else 在"列表为空"时执行——所以没评论时会显示"来抢沙发"。url_for('login') 按函数名生成网址,以后改路径不用改模板。

STEP 6本地常见报错对照

报错/现象原因解决
ModuleNotFoundError: No module named 'flask'没装 flask,或没激活 venv确认 (venv) 在,跑 pip install flask
TemplateNotFound: index.html模板没放在 templates/,或文件名拼错检查目录名必须叫 templates,文件在里面
页面报 jinja2.exceptions模板里 {{ }}/{% %} 写错或没闭合对照本篇模板逐字检查标签配对
端口被占用 Address already in use上次的程序还在跑换端口(改 port=8001)或关掉旧进程
PowerShell 不让激活 venv执行策略限制以管理员运行 Set-ExecutionPolicy RemoteSigned 后重试
发评论报 KeyError: 'text'表单 name 和 request.form["..."] 对不上确认 <textarea name="text"> 与代码一致

STEP 7部署到 VPS:让全世界能访问

本地跑通后,把它搬上服务器。这一节假设你已读完主教程 PART 3:有一台 VPS、能 SSH 登录、做过基础加固。链路和主教程 4.6–4.7 一致,这里给针对本项目的完整命令。

7.1 上线前改两处(安全)

务必改
  1. 换掉 secret_key:把 app.secret_key = "dev-only-..." 改成读环境变量,别让密钥进代码仓库: 
    import os
app.secret_key = os.environ.get("SECRET_KEY", "dev-only-change-me")
    上线时在服务器设置一个真随机值(下面 systemd 里注入)。
  2. debug 关闭:用 gunicorn 启动时不会执行 app.run(debug=True) 那一行(它在 __main__ 里,gunicorn 直接 import 不触发),所以线上自动是非 debug,放心。

7.5 Nginx 反向代理

让外部 80 端口的访问转发给本机 8000 的 gunicorn。新建 /etc/nginx/sites-available/comment-site:

/etc/nginx/sites-available/comment-site 
server {
    listen 80;
    server_name 你的域名.com www.你的域名.com;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
sudo ln -s /etc/nginx/sites-available/comment-site /etc/nginx/sites-enabled/
sudo nginx -t                       # 测语法
sudo systemctl reload nginx

域名解析(A 记录指向服务器 IP)按主教程 4.5 配好。现在访问 http://你的域名.com 应能看到留言板。
