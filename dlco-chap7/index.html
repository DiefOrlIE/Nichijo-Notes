背景 A0：从一行C代码，到CPU里跑的0/1

进入第1讲之前必看

你写了一行 C 代码：

int c = a + b;

它在 CPU 眼里究竟是什么？答案是几条二进制指令。比如在 RISC-V 上大致变成：

lw   x5, 0(x10)   # 从内存把 a 取到寄存器 x5
lw   x6, 4(x10)   # 取 b
add  x7, x5, x6   # x7 = x5 + x6
sw   x7, 8(x10)   # 写回内存里的 c

每一条汇编对应一个 32 位的 0/1 串，这个 0/1 串就是 CPU 真正执行的"机器指令"。

什么是"指令系统"（ISA）

一台 CPU 能识别的所有机器指令的集合 + 它们的编码规则，叫指令集架构（Instruction Set Architecture）。本章就在讲："设计一套 ISA 要考虑哪些事？"

类比：ISA 就像菜谱里"能用的所有动词"——切、炒、煎、煮……配上"宾语写在哪里"（操作数）。厨师（CPU）只认这套词汇表。

本章地图

  • 第1讲：通用设计——一条指令需要哪些字段？怎么编码？有哪几种风格？
  • 第2讲：以 RISC-V 为例，看一套真实 ISA 长什么样。

背景 A1：CPU 内部速览（10秒版）

理解后面所有内容的脚手架

第6章学过的 CPU 关键部件，本章会反复用到，先建立印象：

部件作用
PC程序计数器，指着"下一条要执行的指令在内存哪个地址"
IR指令寄存器，缓存当前正在执行的那条指令
GPRs通用寄存器堆（一组小而快的"草稿纸"，一般 16~32 个）

一条指令的"生命周期"

取指 → 译码 → 取操作数 → 执行 → 写回结果 → 更新PC。这六步是接下来 p4 的主线。

这一讲的所有"设计选择"，都是为了让这六步编码省空间、执行速度快、硬件好实现这三个目标之间找平衡。

指令系统概述

PDF 第 3 页

图里两个小人站在一面"墙"两侧，墙就是指令集。这张图想说一件事：

ISA 是软硬件的"分界面"（interface）

  • 硬件设计者看 ISA：要保证每条指令我都能用电路实现，且实现起来不要太复杂
  • 软件/编译器作者看 ISA：要能把高级语言翻成这些指令，且翻得高效
双方都想要的东西常常矛盾——硬件想简单（指令少），软件想方便（指令多、表达力强）。设计 ISA 就是在调和这个矛盾。

冯·诺依曼对指令的两条铁律

  1. 指令用二进制表示，和数据一起放在主存里（程序即数据）
  2. 一条指令由两部分组成：操作码（干啥）+ 操作数地址（对谁干）

一条指令中应该有几个地址码字段

PDF 第 6 页

地址码数量越少→指令越短；越多→灵活但臃肿。五种选择：

地址数含义典型用途
0只有 OPnop / halt / 栈机的算术指令
1OP A1 — A1 同时做源和目的累加器型 add A：acc ← acc + M[A]
2OP A1 A2 — 结果覆盖其中一个x86 的 add ax, bx：ax ← ax + bx（最常用）
3OP A1 A2 A3 — 两源一目的分开RISC 风格：add rd, rs1, rs2
多SIMD 一次处理多元素向量 / 矩阵指令
三地址指令最"干净"（不破坏源操作数），但指令较长。RISC（包括 RISC-V）选三地址，CISC（x86）选二地址。

指令格式的设计原则

PDF 第 7 页

6条基本原则（解释下含义）

  • 应尽量短——指令占内存，短则代码紧凑
  • 有足够操作码位数——n 位操作码最多 2ⁿ 种指令，要够用
  • 编码唯一——同一串 0/1 不能有两种解释，否则译码歧义
  • 字长是字节整数倍——8/16/32/64 位，方便按字节寻址
  • 地址字段个数合理——见 p6
  • 指令尽量规整——同类指令格式一致，硬件译码电路简单
操作码 + 功能码：一个常见技巧。比如 RISC-V 用 7 位 opcode 区分大类（算术 / 分支 / 访存），同一大类内再用 3 位 funct3 + 7 位 funct7 区分具体操作。后面 p46 起会反复看到。
"LD / ST / INC / BRN 四条已足够编任何程序"是计算理论结论（图灵完备），但程序会非常长。所以实际 ISA 都会提供丰富的指令——是性能与简洁的折中。

寻址方式（Addressing Modes）概述

PDF 第 10 页

核心定义

寻址方式就是"根据指令里的地址码，找到操作数的具体方法"。同一个地址字段值，配上不同的寻址方式，找到的操作数完全不同。

地址码编码三原则（很美）

原则为什么
地址码尽量短指令短→省内存
操作数位置要灵活，地址空间要大编译器才能优化产生高效代码
地址计算过程尽量简单执行快
这三条本质上是"省空间 vs 灵活性 vs 速度"的三角折中。后面看到的 7 种寻址方式，每一种都是在这个三角里站不同位置。

指令本身的寻址简单（PC+1 顺序 / 跳转用操作数寻址），所以"寻址方式"通常专指操作数寻址。

背景 A2：为什么需要这么多种寻址方式

进入 p11 前必看

看下面这几行 C 代码，每一行对操作数的"找法"都不一样：

x = 42;            // 立即数：42 直接写在指令里
x = y;             // 直接 / 寄存器：y 在固定位置
x = *p;            // 间接：p 是地址，要从 p 指的地方取
x = arr[i];        // 变址 / 偏移：arr 首地址 + i×元素大小
x = obj->field;    // 偏移：obj 是基址，field 是偏移
push x;            // 栈：操作数在栈顶，地址默认

每一种语言构造，对应一种"操作数在哪、怎么找到"的模式。如果 ISA 不提供这些寻址方式，编译器就只能用多条指令凑出来——代码会很长、很慢。

所以 p12 那张大表里的 7 种基本寻址方式，每一种都是为某类常见编程场景服务的。看不懂时，回想这里的 C 代码示例。

统一记号

  • A：指令里的地址字段值
  • R：寄存器编号
  • (X)：X 中存的内容
  • EA：Effective Address，有效地址（操作数真正所在的内存地址）

寻址方式的"指定方式"

PDF 第 11 页

两种实现：寻址方式信息放哪？

  1. 没有专门寻址方式位——由操作码本身决定。例如 MIPS / RISC-V：lw 必定是偏移寻址，add 必定是寄存器寻址。
    优点：指令规整，硬件译码电路简单。RISC 喜欢这种。
  2. 有专门寻址方式位——每个操作数前面带几位说明"我用哪种寻址"。例如 x86：mov 后面可以跟立即数、寄存器、内存、内存间接……
    优点：灵活。代价：译码复杂，指令变长。CISC 风格。

有效地址 EA 的含义

EA 是操作数真正所在的存储单元地址（可能是逻辑地址，要经 MMU 翻译成物理地址）。指令字段给的常常不是 EA，要经过寻址方式 + 地址码的计算才能得到 EA。

偏移寻址是"组合拳"：把直接（给常量地址）和寄存器间接（给基址寄存器）合并起来，灵活但稍复杂。下一页拆成三种变体。

偏移寻址三变体：相对 / 基址 / 变址

PDF 第 13 页

偏移寻址 EA = A + (R) 中 R 可以是不同寄存器，于是有三种"味道"：

名字公式用途
相对寻址EA = A + (PC)分支跳转！转移距离与当前指令相对，方便代码搬移
基址寻址EA = A + (B)，B 是基址寄存器访问"以某基址为参考"的全局/段数据
变址寻址EA = A + (I)，I 是变址寄存器，A 是首址循环访问数组：A 不动，改 I 即可
记忆口诀：基址=基不动地址变；变址=地址不动变下标。RISC-V 把基址和变址统一成一种偏移寻址（见 p15）。

指令中显式给出的地址码 A 叫"形式地址"——它不是真地址，要经计算才出 EA。

变址寻址实现线性表元素的存取

PDF 第 14 页

"自动变址"——CISC 的甜点

指令里 A 给数组首地址（如 100），变址器 I 每用一次就自动加/减元素大小 x：

EA = A + (I)，I 自增/自减 x
  • for(i=0;i<N;i++) 用 加（地址低→高）
  • for(i=N-1;i>=0;i--) 用 减（地址高→低）
  • 元素是 1 字节，I 每次 ±1；元素是 4 字节（int），I 每次 ±4
RISC-V / MIPS 不提供自动变址。它们把变址和基址统一成一种偏移寻址，需要程序员/编译器用单独的 addi 指令显式改变寄存器值。代码稍长，但硬件简单、流水线友好。
这是 CISC vs RISC 的一个具体例子：CISC 把"循环访问数组"做成一条指令；RISC 拆成"算下标 + 取数 + 更新下标"几条简单指令。

例题：MIPS32 中的循环处理

PDF 第 15 页

把上一页的概念用真实汇编演一遍。要实现 for(i=0;i<N;i++) g+=A[i];：

          add  $7, $0, $0     ; i = 0
Loop:     add  $10, $7, $9    ; $10 = base + i = &A[i]
          lw   $6, 0($10)     ; $6 = A[i]，0($10) 是偏移寻址
          add  $8, $8, $6     ; g = g + A[i]
          addi $7, $7, 4      ; i += 4（int=4字节，相当于"i++"）
          bne  $7, $2, Loop   ; if i != N×4 跳回 Loop
  • 0($10)：偏移寻址，A=0，R=$10。MIPS 不分基址/变址，统统叫偏移
  • addi $7,$7,4：MIPS 不支持自动变址，所以用专门指令递增下标
  • Loop 标号：汇编器/链接器替你算好相对偏移，不必手动算分支距离
记住：标号是给程序员看的，最终机器码里只有偏移量。链接器通过"重定位"把符号变成实际地址。

每个 float 4 字节，所以 &a[i] = 104 + i×4。i=99 → 104 + 396 = 500。✓

② b[i][j]（short[4][4]，起始 504）

每行 4 个 short = 8 字节，每个元素 2 字节。&b[i][j] = 504 + i×8 + j×2。i=3,j=2 → 504+24+4 = 532。✓

③ d[i]（double[10]，起始 544）

每个 double 8 字节。&d[i] = 544 + i×8。i=9 → 544+72 = 616。✓

这就是编译器实际做的事：把 arr[i][j] 翻译成"基址 + i×行步长 + j×元素大小"。硬件支持复杂寻址方式（如比例变址 EA=B+I×s），编译器就能少生成几条乘法指令。

三个本质洞察

  1. 偏移寻址是 RISC 的主力。它把"内存访问"压成一种统一模式：base + offset。RISC-V 的 lw rd, imm(rs1) 就是这个套路。
  2. 访存次数决定速度。要计算 EA 的方式都至少访存 1 次；间接方式访存 2 次（先取指针、再取数）。
  3. 偏移的三个变体（相对/基址/变址）选择的是 R 是哪个寄存器——R=PC 就是相对，R=B 就是基址，R=I 就是变址。本质是同一个公式 EA=A+(R)。

扩展操作码 — 经典例题

PDF 第 21 页 · 高频考点

题目

16 位指令字，每个地址码 6 位。已有 15 条二地址指令、34 条一地址指令。问零地址指令最多几条？

思路：操作码"由短到长扩展"

  • 二地址：16 - 6 - 6 = 4 位操作码。4 位 = 16 种编码，用掉 15 种，留 1 种（1111）作扩展前缀。
    二地址编码：0000 ~ 1110（15 种）
  • 一地址：操作码扩到 16 - 6 = 10 位，但高 4 位必须是 1111（已用作扩展标记），剩 6 位自由。

    即可用 1111 + (000000~111111) 共 64 种，要用 34 种，留 30 种作再扩展：

    前缀 11110 (00000~11111) = 32 种 + 11111 (00000~00001) = 2 种，共 34 种
  • 零地址：剩下"11111 (00010~11111)"作前缀（共 30 个组合），后面再用 6 位扩展：
    30 × 2⁶ = 1920 条 = 15 × 2⁷
关键技巧：每次扩展时，必须留出至少 1 个高位编码当"标记"，告诉译码器"我后面还有更长的操作码"。否则会产生编码歧义。

背景 A3：标志位是干啥的？

进入 p22 前必看

问题：你在 C 里写 if (a > b) goto L;，CPU 怎么判断 a>b？

答：CPU 没有专门的"比较器"，它用减法来比较：算 a-b，然后看结果的特征。

4 个标志位（条件码 CC）

标志什么时候置1用途
ZF (Zero)结果为 0判等于：a==b → ZF=1
SF/NF (Sign/Negative)结果最高位为 1（即负）判正负
CF (Carry)无符号运算有进位/借位无符号比较
OF/VF (Overflow)带符号运算溢出带符号比较

关键事实

同一个 a-b 运算，硬件同时生成全部 4 个标志。后面的 bgt / blt 等指令只是看哪个组合来决定跳不跳。

比较规则（要背）

  • 无符号 a>b：CF=0 且 ZF=0
  • 带符号 a>b：SF=OF 且 ZF=0
为什么带符号要看 SF==OF？因为带符号减法可能溢出，溢出会"翻转"符号位。SF==OF 表示"符号位是真实的"，再判 ZF 看是不是等于。

条件测试方式（Condition Codes）

PDF 第 22 页

典型套路

sub r1, r2, r3    ; r1 = r2 - r3，同时生成 ZF/CF/SF/OF
bz  label         ; ZF=1 就跳到 label

这正是 C 的 if (z==0) goto label; 编译后的样子。

带符号 vs 无符号

虽然两者运算电路完全一样，但判断大小的标志位组合不同：
  • 无符号 A>B：(ZF=0) ∧ (CF=0)
  • 带符号 A>B：(ZF=0) ∧ (SF=OF)

标志存哪里

常用位置：标志寄存器 / 条件码寄存器 / 状态寄存器 / 程序状态字。RISC-V 与众不同——它没有专门的标志寄存器，比较结果存到通用寄存器里（用 slt），见 p51。

合并优化

有些 ISA 把 cmp+bgt 合成一条 bgt r1, r2, label，减少指令数。RISC-V 就是这种风格。

用 4 位补码演算两个减法。规则：减法 a-b = a + (-b) = a + ~b + 1。

例1：-7 - 6 = -7 + (-6) = +3（理论值）实际算 9 + (-6) = 3

MIPS！ RISC 每条指令短而简单，CPI（每条指令周期数）接近 1；CISC 的 ADDL 单条指令内部要做"两次取数+加+写回"，CPI 很大。

CISC 风格的 6 个特点

PDF 第 31 页
  1. 指令系统复杂：变长操作码 + 变长指令字 + 指令多 + 寻址多 + 格式多
  2. 指令周期长：多数指令需多个时钟周期完成
  3. 各种指令都能访问存储器：不像 RISC 限制于 load/store
  4. 采用微程序控制：复杂指令用微指令实现
  5. 有专用寄存器：累加器、栈指针等有专门用途
  6. 难以编译优化：指令格式多样，编译器难生成最优代码

典型：VAX-11/780（DEC 的小型机，CISC 巅峰之作）

  • 16 种寻址方式、9 种数据格式、303 条指令
  • 一条指令 = 1~2 字节操作码 + N 个操作数说明符
  • 每个说明符 1~10 字节——也就是说一条指令长度可以非常夸张
CISC 的复杂度催生了 RISC 思想（见下页）。

CISC 的缺陷 → RISC 的诞生

PDF 第 32 页

CISC 暴露的问题

  • 指令系统庞大 → 研制周期长、设计正确性难保证、调试维护难
  • 指令操作复杂 → 增加机器周期 → 性能反而下降

关键发现（IBM, 1975）

"20/80 法则"：在程序中，最常用的简单指令占程序 80%，但只占指令集种类的 20%。剩下 80% 的复杂指令几乎不用，却在微程序控存里占 80%。

结论：为这些少用指令付出的硬件代价不值得，不如砍掉。

  • Stanford 的 MIPS（John Hennessy）
  • IBM 的 801（John Cocke，最早提出概念）
这三个人都拿了图灵奖。Patterson 和 Hennessy 是 2017 年的，写过那本经典教材 Computer Organization and Design。

RISC 风格的 6 个特点

  1. 简化的指令系统：指令少 / 寻址方式少 / 格式少 / 指令长度一致

异常和中断分类

PDF 第 35 页

程序执行中可能被"中断"，返回时叫"断点"。事件分两类：

① 内部"异常"（exception）— CPU 内部发生

  • 硬故障中断：电源掉电、硬件线路故障
  • 程序性中断（Exception）：执行某条指令时出意外，比如 溢出 / 缺页 / 越界 / 越权 / 越级 / 非法指令 / 除数为 0 / 堆/栈溢出 / 访问超时 / 断点 / 单步 / 系统调用

② 外部"中断"（interrupt）— CPU 外部发生

通过中断请求信号通知 CPU：实时钟、控制台、打印机缺纸、外设准备好、采样到时、DMA 传输结束……

中文翻译容易混。英文里 exception = 同步事件（指令引起），interrupt = 异步事件（外设引起）。系统调用属于 exception（因为是程序主动用 ecall 触发的）。

异常和中断的处理流程

发生异常/中断后的标准动作：

  1. 当前指令执行到一个安全点
  2. CPU 从用户态切换到内核态（保护现场）
  3. 跳转到 OS 内核的异常/中断处理程序
  4. 处理完后，决定返回方式：
    • 返回当前指令（缺页处理后重试这条 load）
    • 返回下条指令（系统调用做完，继续）
    • abort（致命错误，终止进程）
核心概念：用户进程的正常控制流中"插入了一段内核控制路径"。这是 OS 实现"虚拟化 CPU"的基础——每个进程都觉得自己独占 CPU，其实是被频繁打断的。
这一节其实是操作系统课的内容了，本章只需要知道 ISA 提供哪些机制（ecall/ebreak/CSR 寄存器）支持中断处理，具体细节后面 OS 课会展开。

RISC-V 是什么

UC Berkeley 在 2010 年发起的开源指令集。"V" 是罗马数字 5，表示这是 Berkeley 的第五代 RISC（前面是 RISC-I 到 RISC-IV）。

三大特点

  1. 开源免费：不像 ARM 要付授权费，任何人都能造 RISC-V 芯片
  2. 模块化：基础集 RV32I 只有 47 条指令，需要乘除就加 M 扩展，要浮点就加 F/D 扩展……按需配置
  3. 大道至简：抛弃所有历史包袱，从零设计，硬件实现简单

为什么教材选它

  • x86 太复杂，教学不友好（变长指令、累积几十年的特性）
  • MIPS 设计漂亮但商业上式微
  • ARM 闭源，且 ARMv8 比较复杂
  • RISC-V 是 2020 年后学术界事实标准，国内龙芯也借鉴它（虽然龙芯是 LoongArch）
学懂 RISC-V，看其他 RISC 类 ISA（ARM/MIPS）会很快——它们都遵循相同的设计哲学，只是细节不同。

背景 B1：为什么 RISC-V 要有 6 种指令格式

进入 p43 前必看

RISC 想要"指令尽量规整"，理想是一种格式。但实际上不同指令需要不同字段：

指令类型需要什么字段
寄存器运算 add rd,rs1,rs23 个寄存器号
立即数运算 addi rd,rs1,imm2 个寄存器 + 一个立即数
存数 sw rs2,offset(rs1)2 个源寄存器 + 偏移（没有目的寄存器）
分支 beq rs1,rs2,offset2 个源寄存器 + 跳转偏移（没有目的寄存器）
长立即数 lui rd,imm201 个寄存器 + 20 位立即数
跳转 jal rd,offset1 个寄存器 + 大偏移

所以 RISC-V 设计了 6 种格式 R / I / S / B / U / J，每种针对一类需求。但有个巧妙设计：

关键约束：rs1、rs2、rd 在所有格式里都尽量放在固定位置（同样的 bit 位置），这样译码器可以在还没看完操作码时就开始读寄存器，加快流水线。这是 RISC-V 设计的精髓之一。

这也解释了为什么 S 型和 B 型的立即数被"拆开"——因为必须让 rs1/rs2 保持在固定位置，立即数只能用剩下的零散 bit 位组合而成。

16 位 RISC-V 压缩指令格式（RVC）

PDF 第 44 页

RVC = Compressed，目的：缩短代码量，用一点点时间换大量空间（嵌入式特别在意）。

核心点

  • 16 位长，是 32 位指令的"快捷方式"
  • 每条 16 位指令都对应一条等价的 32 位指令，硬件先把 16 位转 32 位再执行
  • 仍然是 32 位架构、32 个寄存器、处理 32 位数据
  • 有 8 种格式（CR/CI/CSS/CIW/CL/CS/CB/CJ），但只能用部分寄存器（rs1'/rs2'/rd' 只有 3 位，限制访问 x8~x15）
设计巧思：限制只能用最常用的 8 个寄存器，省下 2 个 bit 给立即数。统计上 80% 指令只用这 8 个寄存器，所以代码量能省 25%~30%。
看到 c. 开头的助记符就是压缩指令（c.add、c.lw、c.beqz...）。汇编时混着用：编译器自动把能压缩的换成 16 位的。

U 型指令详解 — lui / auipc

PDF 第 47 页

lui rd, imm20（Load Upper Immediate）

把 20 位立即数放到 rd 的高 20 位，低 12 位清 0。

为什么要 lui？

I 型立即数只有 12 位（范围 -2048~2047），装不下 32 位常数。所以用两条指令：lui 装高 20 位 + addi 装低 12 位。

例：int x = -8191

-8191 的 32 位补码：1111 1111 1111 1111 1110 0000 0000 0001

lui  x5, 1048574    ; 1048574 = 0xFFFFE，装入 x5[31:12]
                    ; → x5 = 0xFFFFE000
addi x5, x5, 1      ; 加 1（imm 符号扩展为 0x00000001）
                    ; → x5 = 0xFFFFE001 = -8191 ✓
观察 lui 后 x5 = 0xFFFFE000——高 20 位是给定值，低 12 位补 0。lui 不做符号扩展（U 型立即数被当成位串直接装入）。

auipc rd, imm20（Add Upper Immediate to PC）

把 20 位立即数加到 PC 高 20 位，结果存 rd。用于实现"PC 相对"的大跳转。auipc x10, 0 可以获取当前 PC 值。

立即数装入易错例 — int x=8191

PDF 第 49 页 · 重点例题

8191 的 32 位机器数：0000 0000 0000 0000 0001 1111 1111 1111

错误方案

lui  x5, 1          ; x5 = 0x00001000
addi x5, x5, -1     ; 期望：0x00001000 + 0xFFF
                    ; 实际：addi 符号扩展 0xFFF = -1
                    ; → x5 = 0x00001000 - 1 = 0x00000FFF = 4095 ✗

问题：低 12 位的最高位是 1（0xFFF=4095，二进制 1111_1111_1111），addi 符号扩展会把它当 -1。

正确方案：先大后小，再调整

lui  x5, 2          ; x5 = 0x00002000 = 8192
addi x5, x5, -1     ; 8192 - 1 = 8191 ✓
规则：因为 addi 总按符号扩展，imm12 范围是 -2048 ~ +2047。如果目标常数的低 12 位首位是 1，必须先 lui 装一个稍大的数，再用 addi（imm12 为负）减下来。
编译器/汇编器会自动处理这事，所以你写汇编时通常用伪指令 li x5, 8191，由汇编器选择正确的 lui+addi 组合。但手动构造时必须考虑这个陷阱。

比较指令例题 — 64 位 long long 加法

PDF 第 51 页 · 高级例题

题目

32 位机器上做 64 位加法 z = x + y。x 的高/低 32 位在 x13/x12，y 在 x15/x14，z 在 x11/x10。

难点：低 32 位的进位怎么传给高 32 位？

RISC-V 没有 CF 标志位，所以要用 sltu 检测进位：

add  x10, x12, x14   ; z低 = x低 + y低
sltu x11, x10, x12   ; 若 z低 < x低，说明加法溢出了 → x11=1（这就是进位！）
add  x16, x13, x15   ; 暂存 x高 + y高
add  x11, x11, x16   ; z高 = (x高 + y高) + 进位

原理

无符号加法溢出判据：a+b 如果产生进位，那么"和" < "加数 a"（因为高位丢了）。所以 sltu (a+b), a 的结果就等于进位 CF。
这是 RISC-V"无标志位"设计的代价：本来 x86 一条 adc（带进位加）搞定的事，RISC-V 需要 4 条指令。但好处是每条指令是无状态的，流水线、乱序执行都更简单。

控制转移类：J / I / B 型

PDF 第 52 页

J 型：jal rd, imm（Jump And Link）

功能：R[rd] ← PC+4；PC ← PC + SEXT[imm[20:1]] << 1

  • rd=x1(ra) → 过程调用（保存返回地址到 ra）
  • rd=x0(zero) → 无条件跳转（不保存返回地址）

I 型：jalr rd, rs1, imm

功能：R[rd] ← PC+4；PC ← R[rs1] + SEXT[imm[12]]

过程调用的返回：jalr x0, ra, 0（不保存返回地址，跳到 ra 指向的地方）。这就是 ret 伪指令。

B 型：beq / bne / blt / bge / bltu / bgeu

条件分支。bltu/bgeu 是无符号比较，blt/bge 是带符号。

转移目标 = PC + SEXT[imm[12:1]] << 1

为什么左移 1 位？ 因为指令地址必须是 2 字节对齐的（RVC 是 2，RV32G 是 4）。所以最低 1 位永远是 0，不用编码，立即数可以多 1 位精度。

题目

int x,y,z 在 x5,x6,x7。实现 z=x+y，检测溢出。

溢出判据（带符号）

"y<0 且 x+y≥x" 或 "y≥0 且 x+y<x" → 溢出。

add   x7, x5, x6           ; z = x + y
slti  x28, x6, 0           ; x28 = (y < 0) ? 1 : 0
slt   x29, x7, x5          ; x29 = (z < x) ? 1 : 0
bne   x28, x29, overflow   ; 若两者不同 → 溢出，跳走

分析

逻辑：x28 表示"y 是负数"，x29 表示"加完结果变小了"。

  • y≥0 时本应让结果变大，结果反而 < x → 溢出（x28=0, x29=1）
  • y<0 时本应让结果变小，结果反而 ≥ x → 溢出（x28=1, x29=0）
  • 两者相同（00 或 11）→ 没溢出

分支偏移量怎么算

假设 overflow 标号离 bne 还有 20 条指令，每条 4 字节 → 偏移 80 字节。但 B 型 imm 已经左移 1 位，所以指令里的 imm = 80/2 = 40 = 0000 0010 1000。

按 B 型编码（imm[12|10:5] + imm[4:1|11]）填入机器码 → 0000010 11101 11100 001 10000 1100011。

B 型立即数拆分顺序是 [12]|[10:5]+[4:1]|[11]——很反直觉，但硬件能并行重组。考试时要会按这个顺序拼。

系统控制类指令

PDF 第 55 页

fence / fence.i — 内存屏障

RISC-V 采用宽松一致性模型（loads/stores 可乱序）。fence 强制规定一个屏障：屏障之前的内存操作必须先完成，之后的才能开始。多核同步必备。

ecall / ebreak — 自陷指令（Trap）

  • ecall：environment call，从用户态主动陷入OS 内核。系统调用（read/write/...）就是用它实现
  • ebreak：environment break，陷入调试环境。GDB 设断点就靠它

csrxxx — CSR 控制状态寄存器

6 条指令（csrrw/csrrs/csrrc/csrrwi/csrrsi/csrrci），用于读写 RISC-V 的 CSR 寄存器（如 mstatus、mepc、mtvec）。这些寄存器存的是 CPU 模式、中断使能、异常入口地址等。

CSR 是操作系统/虚拟化的接口。本课不展开，OS 课会详细讲。考试只需知道这 6 条 csr 指令的存在和大致功能。

64 位架构指令举例

PDF 第 57 页

题目

把 32 位常数 0x003D0500 装入 64 位寄存器 a0。

常数二进制：00000000 00111101 00000101 00000000

分析

  • 第 31~12 位：00000000 0011 1101 0000 = 976
  • 第 11~0 位：0101 0000 0000 = 1280

解

lui  a0, 976        ; a0 高 20 位装入 976
                    ; a0 = 0x003D0000，高 32 位按符号扩展（这里是 0）
addi a0, a0, 1280   ; a0 = 0x003D0500 ✓
64 位 lui 的细节：lui 装入 32 位寄存器的高 20 位+低 12 位补 0；64 位时把这 32 位结果符号扩展为 64 位。所以这里第 31 位是 0，高 32 位也全 0。如果第 31 位是 1，要小心高 32 位会变成全 1。

n 位 × n 位的乘积是 2n 位。三种 ISA 各有处理方式：

ISA乘法做法
IA-32一个源在累加器 AL/AX/EAX，结果存 AX (16位)/DX-AX (32位)/EDX-EAX (64位)
MIPSmult 把 64 位乘积放到专门的 Hi/Lo 两个寄存器；溢出判据：Hi 每一位都等于 Lo 的最高位

思考题答案

PDF 第 61 页

(1) 溢出判断规则

  • 无符号：乘积高 n 位非 0 → 溢出
  • 带符号：乘积高 n 位每一位都等于乘积低 n 位的符号位 → 不溢出；否则溢出

带符号判据等价说："高位是低位的符号扩展"才合法，否则真实值超出 n 位带符号范围。

(2) RISC-V 实现（4 条指令）

mul   t0, a0, a1    ; 低 32 位乘积 → t0
mulh  a0, a0, a1    ; 高 32 位乘积 → a0
srai  t0, t0, 31    ; 低 32 位算术右移 31 位 → 全 0 或全 1（取低 32 位符号位扩展）
xor   a0, a0, t0    ; 若高 32 位 == 低 32 位符号扩展 → xor=0 → 不溢出
巧妙之处：srai 31 把低 32 位变成"全 0 或全 1"，正好是判据里需要的"符号扩展"形式。xor 结果非 0 即溢出。整个判断 4 条指令、无分支、流水线友好。

整数除运算的细节

RISC-V 除法不抛异常

右图实测：C 程序里 1/0 不会崩，而是返回 -1。原因：RISC-V 的 div 在硬件层面对两种异常情况有明确规定：

情况div 结果rem 结果
除以 0商 = 全 1 = -1（带符号视角）余 = 被除数
最小负数 ÷ -1（溢出）商 = 被除数（最小负数）余 = 0

设计权衡

好处：简化硬件，所有除法都返回一个值，不需要中断处理。
坏处：程序员/编译器必须自己检测除零和溢出，否则得到"错误但能跑"的结果。

右下角的截图证实了这个设计：printf("res = %d\n", 1/0) 在 riscv64-gcc 下输出 res = -1，没有崩溃。

这是 RISC-V 一以贯之的哲学："简单优先"——硬件只做必须做的事，复杂判断丢给软件。和 p51 的"无标志位用 sltu 算进位"、p58 的"乘除不检测溢出"完全一脉相承。
