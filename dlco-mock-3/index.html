A　运算方法与运算部件

一、运算部件（加法器 · 标志 · ALU）

3.1易填空
带标志加法器：溢出标志 OF = ______（带符号用）；进位/借位标志 CF = ______（无符号用）。
参考答案
Cₙ ⊕ Cₙ₋₁；Cout ⊕ Cin。另有 ZF（结果全 0）、SF（结果最高位）。
3.2易填空
先行进位（CLA）定义两个辅助信号：生成 Gᵢ = ______，传播 Pᵢ = ______。
参考答案
Aᵢ·Bᵢ；Aᵢ⊕Bᵢ（或 Aᵢ+Bᵢ）。用 Cᵢ=Gᵢ+PᵢCᵢ₋₁ 展开，使进位并行产生。
3.3易填空
ALU 用 ______ 信号选择执行的操作；运算器里一切运算最终都归结为 ______ + 移位器 + 多路选择器 + 控制逻辑。
参考答案
ALUop；加法器。

二、定点数运算

3.4易填空
补码减法 A − B 的硬件实现：把 B ______ 后送进加法器与 A 相加。
参考答案
取反加一（即加 [−B]补）。减法统一为加法，这是补码的核心优势。
3.5易判断
补码加减法中，"最高位进位与次高位进位不同"（Cₙ⊕Cₙ₋₁=1）即发生溢出。（对／错）
参考答案
对。等价判据：同号相加结果符号反了。
3.6中计算
用双符号位（变形补码）做补码加法，结果两符号位为 10 表示 ______；01 表示 ______；00/11 表示 ______。
参考答案
负溢出；正溢出；无溢出。两符号位不同即溢出，正确结果符号取最高那位。
3.7易解释
原码加减法为什么说"求差永不溢出"？
参考答案
原码求差是两数数值位相减，结果绝对值不会超过较大的被减数，仍在表示范围内，故不溢出。（求和才可能溢出）
3.8中解释
Booth 一位乘法的判断规则是什么？（看哪两位、怎么决定加减）
参考答案
乘数最低位后补 y₋₁=0，从低位起看相邻两位 (yᵢ, yᵢ₋₁)，按 yᵢ₋₁ − yᵢ：01→+X、10→−X、00/11→+0，每步部分积算术右移一位。
3.9中填空
Booth 两位乘法每次看 ______ 位窗口、处理乘数两位，循环 ______ 次；系数取值 {−2…+2}，所以补码两位乘永远不会出现 ______。
参考答案
三；n/2；+3X（只需 0、±X、±2X）。+3X 是原码两位乘才有的麻烦，靠 −X+4X 拆解。
3.10难计算
用补码两位 Booth 乘法计算：[X]补 = 1101，[Y]补 = 0110，求 [X×Y]补。写出每步。
参考答案
[X×Y]补 = 1110 1110（= −18）。 X=−3，Y=6；备 [−X]补=0011、2[X]补=11010、2[−X]补=00110；n=4 → 2 次循环。 
P=00000  Y=0110  y-1=0   初始
 看 y1y0y-1=100 → +2[-X]=+00110
 P=00110
 →算术右移2位: P=00001 Y=1001 y-1=1
 看 y3y2y1=011 → +2[X]=+11010
 P=11011
 →算术右移2位: P=11110 Y=1110
拼接取低8位 = 1110 1110 = −18  （验证 −3×6=−18 ✓）
3.11易判断
定点除法的"不恢复余数法（加减交替法）"比"恢复余数法"快。（对／错）
参考答案
对。余数为负时不必先加回再移位，直接下一步改做加法，省去恢复步骤。

三、浮点数运算（IEEE 754）

3.12中解释
浮点加减法的五个步骤按顺序是什么？对阶时为什么要"小阶向大阶看齐"？
参考答案
对阶 → 尾数加减 → 规格化 → 舍入 → 判溢出。小阶的尾数右移去对齐大阶，因为若让大阶尾数右移会丢掉高位有效数字、损失更大。
3.13中计算
浮点加法：A = 1.01₂ × 2¹，B = 1.10₂ × 2⁻¹，求 A + B（写出二进制结果与十进制值）。
参考答案
对阶：B 右移 2 位 → 0.0110×2¹；尾数加：1.0100+0.0110 = 1.1010×2¹（已规格化）= 11.01₂ = 3.25。验证：A=2.5，B=0.75，和=3.25 ✓。
B　指令集架构（ISA）

四、ISA 基本概念与寻址方式

3.14易填空
ISA 是软件与硬件之间的 ______；它规定指令格式、操作数类型、寄存器组织、寻址方式、______ 等，但不规定具体电路实现。
参考答案
契约 / 分界面；大小端（存储编址方式）。
3.15易解释
ISA 与微体系结构（microarchitecture）有什么区别？
参考答案
ISA 是规范/契约（程序员可见的接口）；微架构是 ISA 的具体硬件实现。同一 ISA 可有多种微架构（如不同性能的处理器）。
3.16中计算
求有效地址 EA：寄存器间接寻址 EA = ______；基址/偏移寻址 EA = ______；间接寻址 EA = ______。
参考答案
R[r]；R[r] + 偏移量；M[指令给的地址]（再访存一次取地址）。
3.17易填空
寻址方式中，最快且不访存的是 ______ 寻址；需要两次访存的是 ______ 寻址。
参考答案
寄存器（直接）；间接。立即寻址操作数在指令里、也很快。
3.18易解释
操作码编码中，定长操作码和扩展（变长）操作码各有什么特点？
参考答案
定长：所有指令操作码位数相同，译码简单但浪费编码空间；扩展：高频指令给短码、低频给长码，提高编码效率（类哈夫曼思想）。

五、指令设计风格与 RISC-V

3.19易填空
操作数风格中，"运算指令只能访问寄存器、仅 Load/Store 访存"的是 ______ 型，它是 ______（RISC／CISC）的基础。
参考答案
装入/存储（load-store）型；RISC。另有累加器型、堆栈型、通用寄存器型。
3.20易判断
RISC 指令定长、寻址方式少、多数单周期；CISC 指令变长、多数需多个时钟周期。（对／错）
参考答案
对。RISC 代表：RISC-V、ARM、MIPS；CISC 代表：x86。
3.21易解释
异常（exception）和中断（interrupt）有什么区别？
参考答案
异常来自 CPU 内部（除零、地址越界、非法指令）；中断来自 CPU 外部（I/O 设备、定时器）。两者都打断指令流、保存现场→处理→恢复。
3.22易填空
RISC-V 指令定长 ______ 位，采用 ______ 访存风格，共有六种指令格式：R / I / S / B / U / J。
参考答案
32；load-store。
3.23易填空
RISC-V 格式判别：寄存器-寄存器运算（add）用 ______ 型；store（sw）用 ______ 型；条件跳转（beq）用 ______ 型；长立即数（lui）用 ______ 型；无条件跳转（jal）用 ______ 型。
参考答案
R；S；B；U；J。load（lw）和短立即数运算（ori）用 I 型。
3.24中解释
为什么 B 型、J 型指令的立即数最低位恒为 0？
参考答案
跳转目标按 2 字节对齐，偏移量一定是偶数，最低位必为 0，所以不占编码位（恒补 0），把省下的位用来扩大跳转范围。
3.25中填空
立即数扩展：immI 是 ______ 扩展（12 位）；immU 是 ______（低 12 位补 0）；immS 由 ______ 两段拼接后符号扩展。
参考答案
符号扩展；低 12 位补 0（非符号扩展）；Instr[31:25] 与 Instr[11:7]。
3.26难计算
RISC-V 立即数拼接。immB 拼接规则 = {Instr[31], Instr[7], Instr[30:25], Instr[11:8], 1'b0}。已知一条 beq 指令机器码（分组）：
0000000 00010 00011 000 01000 1100011
求该分支的偏移立即数 immB（十进制）。
参考答案
immB = +8（跳转到 PC+8）。 字段：Instr[31]=0、Instr[7]=0、Instr[30:25]=000000、Instr[11:8]=0100。 
immB = 0  0  000000  0100  0
       ↑  ↑   ↑       ↑    ↑
     [12][11][10:5]  [4:1] [0]
     = 0 0000 0000 1000(2) = +8
C　CPU 设计

六、CPU 组成与性能分析

3.27易填空
CPU = ______ + ______；其中数据通路是 ______ 逻辑（干活），控制器是 ______ 逻辑（指挥）。
参考答案
运算器 + 控制器；组合；时序。所以说"CPU = 放大版有限状态机"。
3.28易解释
一条指令的执行过程包含哪些主要阶段？
参考答案
取指 → 译码 → 取操作数（读寄存器）→ 执行（ALU）→ 访存（load/store）→ 写回 → 更新 PC。
3.29易填空
（必考）CPU 性能基本公式：CPU 时间 = ______ × ______ × ______。
参考答案
指令条数 × CPI × 时钟周期。
3.30中计算
某程序指令数 1×10⁶ 条，CPI = 2.5，时钟频率 2 GHz，求 CPU 执行时间。
参考答案
周期 = 1/2GHz = 0.5ns；时间 = 10⁶ × 2.5 × 0.5ns = 1.25×10⁶ ns = 1.25 ms。
3.31中计算
MIPS = ______（用主频和 CPI 表示）；若主频 2 GHz、CPI = 4，则 MIPS = ______。
参考答案
主频 / (CPI × 10⁶)；2×10⁹ / (4×10⁶) = 500 MIPS。
3.32易判断
只用主频（或只用 MIPS）就能正确比较两台机器的性能。（对／错）
参考答案
错。必须看 指令数 × CPI × 时钟周期 三要素，单看主频或 MIPS 会被指令数/CPI 差异误导。

七、单周期 CPU 数据通路

3.33难分析
单周期数据通路：写出 lw 指令依次经过的部件，并给出关键控制信号（RegWr / ALUBSrc / MemtoReg / MemWr）的取值。
参考答案
通路：取指 → 读寄存器 R[rs1] → ALU 算地址 R[rs1]+immI → 读数据存储器 → 写回 rd。
控制信号：ALUBSrc=1（B 端选立即数）、ALUctr=add、MemtoReg=1（写回值来自存储器）、RegWr=1、MemWr=0、Branch=0/Jump=0。 lw 是关键路径最长的指令，决定单周期时钟周期。
3.34难分析
单周期数据通路：beq 指令怎么走？下地址逻辑如何决定下一条 PC？给出 RegWr / Branch 取值。
参考答案
通路：读 R[rs1]、R[rs2] → ALU 做减法判零得 Zero → 不写回（RegWr=0）。
下地址逻辑：Branch=1 且 Zero=1 → PC ← PC + immB（分支目标）；否则 PC ← PC+4。 注意 immB 拼接时末位已补 0（=已 ×2，就是字节偏移），目标 = PC+immB，别再额外 ×2。jal 则 Jump=1 选跳转目标、并把 PC+4 写回 rd。
3.35易填空
单周期 CPU 的 CPI = ______；其时钟周期由最长指令 ______ 的关键路径决定。
参考答案
1；lw。
3.36易解释
单周期 CPU 的主要缺点是什么？多周期 CPU 如何改进？
参考答案
缺点：简单指令也被迫使用同一个长周期（按最慢的 lw 定），浪费时间。多周期：把指令拆成多个短阶段、不同指令用不同周期数，且功能部件可跨周期复用。

八、多周期与流水线（了解）

3.37中计算
多周期 CPI = Σ(指令占比 × 周期数)。若 Load 22%×5 + Store 11%×4 + R 型 49%×4 + Branch 16%×3 + Jump 2%×3，求 CPI。
参考答案
1.1 + 0.44 + 1.96 + 0.48 + 0.06 = 4.04。
3.38易填空
经典五级流水线：IF → ______ → EX → ______ → WB。
参考答案
ID（译码/读寄存器）；MEM（访存）。
3.39中解释
流水线的三种冒险分别是什么？数据冒险怎么解决？
参考答案
结构冒险（部件争用，靠指令/数据 Cache 分离）、数据冒险（前指令结果未写回就被用，靠转发 Forwarding；load-use 需阻塞 1 拍 + 转发）、控制冒险（转移目标未定取错指令，靠分支预测）。
3.40易填空
控制冒险的常用解决办法之一是 ______（写出一种即可）。
参考答案
分支预测（静态/动态）／插入 NOP 或硬件清空错误指令／延迟分支，任一。

九、查漏补缺

3.41易填空
（真题填空）冯·诺依曼结构的五大部件是：运算器、控制器、______、______、______。
参考答案
存储器、输入设备、输出设备。CPU = 运算器 + 控制器。
3.42难计算
（真题填空·综合,信息表示 × RISC-V）小端存储，地址 0100H–0103H 依次存 63H、73H、61H、69H。
(1) 当无符号整数，右移 24 位后的十进制值 = ______；
(2) 当字符串（a=61H），其内容 = ______；
(3) 当单精度浮点数，阶码 E 的值 = ______，真实指数 = ______；
(4) 当一条 B 型指令，所含立即数 immB = ______。
参考答案
小端 → 整数 = 0x69617363，二进制 0110 1001 0110 0001 0111 0011 0110 0011。
(1) >>24 取最高字节 = 0x69 = 105；
(2) 按地址逐字节读：63='c',73='s',61='a',69='i' → "csai"（字符串与端序无关）；
(3) S=0；E = bit[30:23] = 1101 0010 = 210；真实指数 = 210 − 127 = 83；
(4) immB = {b31, b7, b[30:25], b[11:8], 0} = {0,0,110100,0011,0} = +1670。 这就是去年填空第 2 题——一段字节按 4 种规则解读，是本课最具代表性的综合题。
3.43中计算
（真题综合）对一个给定的 opcode：R 型指令理论上最多有多少条？I 型呢？（用字段宽度算）
参考答案
R 型靠 funct7(7位)+funct3(3位) 区分 → 最多 2¹⁰ = 1024 条；I 型只靠 funct3(3位) 区分 → 最多 2³ = 8 条。移位类 I 型指令会再借 funct7 的高位细分，是特例。
3.44中计算
（真题分析）jal 的立即数为 imm[20:1]（拼接后含末位 0，共 21 位有效、最高位符号位）。该指令的转移目标地址范围相对 PC 是多少？
参考答案
有效偏移是 21 位有符号数（imm[20:0]，含末位 0），范围 [−2²⁰, +2²⁰−2] 字节，即约 ±1 MB（PC ± 2²⁰ 字节）。B 型分支同理但只有 13 位 → ±4 KB。记住：J 型 ±1M、B 型 ±4K。
