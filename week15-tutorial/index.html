四块拼图：从「有没有答案」到「怎么找异常」

先分清监督 / 无监督，再从无监督里挑三件常用工具：聚类分组、关联规则挖共现、异常检测找离群。读完每个公式你都能自己手算一遍。

0 · 全景地图

先搭骨架，再填血肉

机器学习按「训练数据里有没有标准答案（标签）」分两大类。本次作业的四块，除了第 1 节的对比，其余全落在无监督这一侧：

大类有没有标签在干什么本次涉及
监督学习有 (x, y)学一个能预测标签的映射第 1 节
无监督学习只有 x把样本分组 → 聚类第 2–6 节
挖共现规律 → 关联规则第 7–10 节
找离群点 → 异常检测第 11–12 节
一句话直觉：监督学习是「老师给了答案，你学着对答案」；无监督学习是「没答案，自己看数据长什么样」。聚类看「谁和谁像」，关联规则看「什么和什么一起出现」，异常检测看「谁不合群」。

1 · 监督学习 vs 无监督学习

三个角度：数据长什么样 · 靠什么信号学 · 能干什么活

数据形式与目标

监督学习的每个样本是「特征 + 标签」对 (x, y)。x 是输入（如邮件词频），y 是标准答案（如「垃圾/正常」）。目标：学映射 f : x → y，对没见过的新样本也能预测标签。

无监督学习的样本只有特征 x，没有任何标签。目标退而求其次：发现数据自身的结构——哪些点抱团、哪些维度重要、什么总一起出现、谁是异类。

学习信号（最易混的点）

「学习信号」= 模型靠什么反馈来调整自己，这是两类学习的本质分界。

监督：外部误差
信号来自标签。预测出 ŷ，和真值 y 比，差距由损失函数量化（分类用交叉熵，回归用均方误差），最小化损失来调参。每个样本都有明确「对/错」依据。
无监督：内在准则
没标签可对照，信号来自数据自身准则：同簇点应尽量近（簇内距离）、压缩后还原误差要小、投影后方差要大。衡量的是「结构合不合理」而非「预测对不对」。

常见任务场景

类型任务例子
监督分类（标签离散）垃圾邮件识别、图像分类、违约判断
回归（标签连续）房价、气温、销量预测
无监督聚类用户分群、文档主题归类
降维可视化、特征压缩（PCA、t-SNE）
关联规则购物篮分析、学习行为模式
异常检测欺诈识别、设备故障预警
记忆锚点：有标签→学预测（监督）；无标签→找结构（无监督）。后面三节全是「找结构」的具体招式。

2 · 聚类：距离与目标函数

把没标签的点分成 K 组，组内尽量像、组间尽量远

聚类要量化「像不像」，二维下用欧氏距离（直线距离）：

d(p, q) = √[ (pₓ−qₓ)² + (p_y−q_y)² ]两点欧氏距离

K-Means 想最小化簇内平方误差和 SSE：每个点到所属簇心距离的平方求和，越小越紧凑。

SSE = Σ_k Σ_{x∈C_k} ‖x − c_k‖²越小越好；c_k 是第 k 个簇心
关键性质：这个 SSE 是非凸的，算法只能保证收敛到局部最优，不保证全局最优。这是后面「初始化敏感」一切问题的根源。

3 · K-Means 的四步循环

分配 → 求均值更新 → 重复
  1. 初始化：选 K 个初始簇心（本课程固定取前 K 个点，保证答案唯一）。
  2. 分配：每个点算到 K 个簇心的距离，归给最近的簇；距离相等时归编号小的簇心。
  3. 更新：每个簇的新簇心 = 簇内所有点坐标的平均值。
  4. 重复 第 2、3 步，直到做满 T 次迭代。
c_新 = ( (x₁+…+x_m)/m , (y₁+…+y_m)/m )K-Means 更新：簇内坐标均值，m 为簇内点数

对应作业要补的那行代码：labels==i 取出第 i 簇全部样本，.mean(axis=0) 沿样本方向逐列求均值。

def update_centroids(X, labels, k):
    n_features = X.shape[1]
    centroids = np.zeros((k, n_features))
    for i in range(k):
        centroids[i] = X[labels == i].mean(axis=0)   # 第 i 簇按维度求均值
    return centroids

4 · K-Centroid（medoid）的区别

和 K-Means 只差「更新簇心」这一步

K-Means 的簇心是算出来的虚拟平均点（可能不在数据里）；K-Centroid 要求簇心必须是簇内某个真实样本点。

选法：对簇内每个点，算它到簇内其余所有点的距离之和，选距离和最小的那个点（并列取输入编号最小）。

c_新 = arg min_{p∈C} Σ_{q∈C} d(p, q)簇内到他人总距离最小的真实点
K-MeansK-Centroid
簇心是什么坐标均值（虚拟点）簇内某个真实样本点
怎么算求平均，O(m)两两距离和取最小，O(m²)
抗异常值弱（被极端点拉走）强（中位点式，更稳健）

5 · 初始化的影响 & 空簇处理

为什么固定取前 K 个点

初始簇心选择会直接改变最终结果，因为 SSE 非凸、只收敛到局部最优：

  • 初始中心不同 → 可能收敛到不同的簇划分，结果不唯一；
  • 初始中心选得差（几个落在同一真实簇、或彼此太近）→ 易陷差的局部最优、产生空簇、收敛变慢；
  • 缓解：多次随机重启取 SSE 最小的一次，或用 k-means++（让初始中心尽量分散）。

本课程「固定取前 K 个点」正是为了消除随机性、保证答案唯一可判分。

空簇处理：某次迭代若某簇一个点都没分到，它的簇心保持不变（代码里就是 if n==0: continue 跳过更新）。

6 · 手算走一遍（聚类例题）

6 点，K=2，T=2，K-Means

初始簇心 = 前 2 个点：c₁=(0,0)，c₂=(0,2)。

迭代 1 · 分配

点到 c₁(0,0)到 c₂(0,2)归簇
p1 (0,0)02簇1
p2 (0,2)20簇2
p3 (1,1)√2≈1.41√2≈1.41簇1（并列取小号）
p4 (8,8)≈11.3110.00簇2
p5 (9,8)≈12.04≈10.82簇2
p6 (8,9)≈12.04≈10.63簇2

更新：簇1={p1,p3} → c₁=(0.5, 0.5)；簇2={p2,p4,p5,p6} → c₂=((0+8+9+8)/4, (2+8+8+9)/4)=(6.25, 6.75)。

迭代 2 · 分配

用新簇心：p1、p2、p3 离 (0.5,0.5) 明显更近 → 簇1；p4、p5、p6 离 (6.25,6.75) 更近 → 簇2。簇心几乎不动，结束。

最终标签：p1,p2,p3 → 簇 1；p4,p5,p6 → 簇 2。与标准答案一致 ✓
实现小坑：你代码是「先循环 T−1 次更新、最后再分配一次得标签」。和「分配+更新各做 T 次」殊途同归——标签总由最后一次分配决定，最后那次更新算了也没用上，结果完全一致。

7 · 关联规则：基本概念

从「购物篮」里挖「买 A 的人也常买 B」
  • 事务 (transaction)：一条记录，是一组 item 的集合。如 S1={看视频, 做测验, 看讨论区}。
  • item（项）：事务里的单个元素。如「看视频」。
  • 项集 (itemset)：若干 item 的集合；含 k 个的叫 k-项集。
  • 关联规则：形如 X → y，「出现 X 时往往也出现 y」。X 非空，y 是单个 item 且 y∉X。

8 · 支持度与置信度

一个量「多常见」，一个量「多可靠」

支持计数 = 包含某项集的事务数。支持度 support = 支持计数 / 总事务数。

support(Z) = (包含 Z 的事务数) / (总事务数 N)衡量组合有多常见

置信度 confidence = 出现 X 的前提下也出现 y 的比例，即条件概率 P(y|X)：

confidence(X → y) = support_count(X ∪ {y}) / support_count(X)衡量规则有多可靠

满足 support ≥ minsup 的项集叫频繁项集；同时满足 support ≥ minsup 且 confidence ≥ minconf 的才是要找的强关联规则。

避坑·整数比较：判置信度别用除法（有浮点误差）。把 sc/sx ≥ C/100 两边乘开成纯整数比较：
100 × support_count(X∪{y}) ≥ C × support_count(X)

9 · Apriori 原理（精髓）

用一条性质把 2ᴹ 种组合大幅剪枝
反单调性 / 向下封闭
项集若频繁，其所有子集也一定频繁；反过来，项集若不频繁，其所有超集也一定不频繁。
直觉：超集出现次数不可能比子集多（加限制只会更难满足），子集都不够格，超集更不可能够格。

由此得到逐层算法：

  1. 找出所有频繁 1-项集。
  2. 由频繁 k-项集两两连接，生成候选 (k+1)-项集（前 k−1 项相同才连接）。
  3. 剪枝：候选的任一 k-项子集若不频繁，直接丢掉（不用扫库）。
  4. 扫一遍数据库给剩下候选计数，留 support ≥ minsup 的。
  5. 重复直到生不出新频繁项集；最后从每个频繁项集拆规则、验证置信度。

11 · 异常检测一：相对密度法

看「比邻居稀疏多少」（局部、无分布假设）

这是 LOF（局部离群因子）的简化版。核心：一个点异不异常，不看它绝对有多孤立，而看它比周围邻居孤立多少。

① K 近邻

点 i 的 K 近邻 = 除自己外、欧氏距离最近的 K 个点。距离相同时编号小的优先（排序键用 (距离, 编号) 自动满足）。

② 局部密度

这里的「密度」其实是到 K 个近邻的平均距离。注意方向：值越大 = 邻居越远 = 越稀疏 = 越可能异常。

density(i) = (1/K) · Σ_{j∈KNN(i)} d(i, j)K 个近邻的平均距离（越大越稀疏）

③ 相对异常得分

score(i) = density(i) / avg_neighbor_density(i)自己的稀疏度 ÷ 邻居们的平均稀疏度

分母是「点 i 的 K 个近邻各自 density 的平均」。score > 1 说明 i 比邻居更稀疏，越大越异常。

为什么要「相对」：有的点本就在稀疏区，但邻居一样稀疏，它并不异常——绝对密度会冤枉它。除以邻居密度后，「大家都稀疏」被约掉，只有显著比周围孤立的点得分才高。

手算（8 点，K=2，Q=3）

点 8 (30,30)：2 近邻是点 6、7，距离都是 √((30−10)²+(30−11)²)=√761≈27.59 → density(8)=27.59。点 6、7 的 density≈1.21（紧挨 5/6/7 那团）。score(8)=27.59/1.21≈22.85，远超 1，最异常 ✓。（点 6、7 得分≈1.09，并列取小号 6 在前，输出 8 / 6 / 7。）

12 · 异常检测二：z-score 法

看「偏离均值几个标准差」（全局、假设正态）

假设数据来自正态分布。z 分数把每个点标准化：

z = (x − μ) / σμ 均值，σ 标准差

经验法则：正态下约 95% 的数据落在 ±2σ 内、99.7% 落在 ±3σ 内。所以 |z| > 2（或更严的 3）就当异常。

样本 σ 还是总体 σ：样本标准差分母用 n−1（无偏，ddof=1），总体标准差用 n。两者数值略不同，但本题结论不变——都只标出一个异常点。

手算：data = [1,2,2,1,10,2,1,20]

  • 均值 μ = (1+2+2+1+10+2+1+20)/8 = 39/8 = 4.875
  • 偏差平方和 = 324.875；样本 σ = √(324.875/7) ≈ 6.81（总体 σ=√(324.875/8)≈6.37，结论一样）
  • 各 z：1→−0.57，2→−0.42，10→0.75，20→2.22

只有 20（第 8 个点） 的 |z|≈2.22 > 2，被标记为异常；10 的 |z|≈0.75，没超阈值，不算异常。

masking（掩盖）效应：异常值 20 自己把 μ、σ 抬高了，把同样偏离主群的 10「盖」住，使它显不出异常。这是 z-score 的天生短板。多异常场景改用更稳健的中位数 / MAD。

两种方法对比

相对密度法z-score 法
视角局部（和邻居比）全局（和全体均值比）
分布假设无假设正态
优点发现局部离群、适应密度不均简单、可解释
缺点要算 KNN，O(N²) 偏慢易被异常值污染（masking）
