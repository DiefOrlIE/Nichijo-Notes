整章地图 · 第 9 章在干嘛

补充背景 · 在跳进任何技术细节前必看

这一章的核心目标只有一句话：

模型存在性定理：只要一个公式集合 Ψ 在语法层面看起来自洽（即所谓 Hintikka 集），那么 Ψ 一定真有一个模型让它全为真。

它是后续完备性定理（⊨ ⇒ ⊢）的关键引擎。完备性的证法套路是反证：假设 Γ ⊬ A，把 Γ ∪ {¬A} 系统性地扩张成 Hintikka 集，再用本章方法造一个模型 → 就得到 Γ ⊭ A 的反例 → 矛盾。

本讲流程（按顺序）

  1. 序列数 ⟨a₀,…,aₙ⟩：把有限自然数序列编成一个自然数。（编码工具）
  2. Gödel 码 #X：把语言里的项 / 公式 / 符号都编成自然数。（语法 → 算术）
  3. Hintikka 集：定义一种"分解封闭"的公式集合。（语法上的自洽）
  4. 项模型 ℍ：用项的等价类当论域，造出一个具体模型。（语法变语义）
  5. 引理 3.41：归纳证明这个 ℍ 满足整个 Ψ。（兑现承诺）
流程图：序列数 → Gödel 码 → Hintikka 集 ⇒ 项模型 ℍ ⊨ Ψ

与第 8 章的承接 · 完备性证明在哪一步

补充背景

第 8 章你已经学完了：

  • 形式系统（公理 + 推理规则）+ 推导 Γ ⊢ A 的定义
  • 可靠性：Γ ⊢ A ⇒ Γ ⊨ A（语法对就是语义对）
  • 完备性：Γ ⊨ A ⇒ Γ ⊢ A（语义对必有语法证明）—— 只给了陈述，未证

完备性的标准证法（Henkin 风格）逻辑结构如下：

步骤属于
① 反设 Γ ⊬ A，则 Γ ∪ {¬A} 一致第 8 章（下一讲会讲）
② 把一致集扩张成 Hintikka 集 Ψ下一讲
③ Hintikka 集必有模型（本章核心）第 9 章
④ ⇒ Γ ∪ {¬A} 可满足 ⇒ Γ ⊭ A，与前提矛盾合成

所以第 9 章只负责其中的 ③ —— "Hintikka 集 → 模型"这一段。其它环节下一讲补。

为什么要序列数和 Gödel 码

补充背景

本章看似从一个奇怪的地方（素数乘积）开始，要先理解动机：

  • 一个项可能像 f(g(x, h(y)), c) 这样层层嵌套，结构是一棵树。
  • 我们要把整个树编成一个自然数，而且能机械地还原回原树。

Gödel 的天才一招：用素数指数携带信息。

⟨a₀, a₁, …, aₙ⟩ ≜ 2a₀ · 3a₁ · 5a₂ · 7a₃ · …

由算数基本定理（每个 N>1 唯一分解为素数乘积），这个数和这个序列是一一对应的。

这套技术是 Gödel 不完备定理的奠基技巧。本章只用它做"语法对象 → 自然数"的机械编码，铺垫"哪些元定理是可计算 / 递归可枚举的"这一类深问题。
类比：JSON 序列化 ↔ 这里的 Gödel 码 #。区别是：JSON 用文本作为承载，Gödel 码用一个自然数作为承载。

定义 3.26 · 序列数

PDF 第 2 页

定义

⟨a₀, …, aₙ⟩ ≜ ∏i=0n Piaᵢ，其中 Pᵢ 是第 i 个素数（P₀=2, P₁=3, P₂=5, P₃=7, …）

白话翻译

把一串自然数 (a₀, a₁, …, aₙ) 装进"素数指数"里。一个序列对应一个具体的乘积数。

具体例子

  • ⟨3⟩ = 2³ = 8
  • ⟨3, 1⟩ = 2³ · 3¹ = 24
  • ⟨3, 1, 0, 2⟩ = 2³ · 3¹ · 5⁰ · 7² = 8 · 3 · 1 · 49 = 1176
  • ⟨0, 0⟩ = 2⁰ · 3⁰ = 1
右侧的素数表（前 168 个素数）说明这个编码无论序列多长都有素数可用——素数有无穷个。
注意 ⟨a₀⟩ 和 a₀ 不是同一个数。⟨3⟩=8，不是 3。"⟨·⟩" 是一个把序列变成单一数的函数。

算数基本定理（铺垫）

PDF 第 3 页

定理

给定 ∀N ∈ ℕ 且 N > 1，N 可以唯一分解为有限个素数的乘积：N = p₁a₁ × … × pₙaₙ。

白话

"每个大于 1 的自然数都有独一无二的素因子分解。" 这是序列数能成立的全部基础——只有唯一性才让"从数还原序列"是良定义的。

证明 · 存在性

对 N 做强归纳。若 N 是素数，结束。若 N 不是素数，则 N = a · b，其中 1 < a, b < N。由归纳假设 a, b 都能写成素数乘积 → N 也能。

证明 · 唯一性

反证法。假设 N 有两种分解：

N = p₁a₁ p₂a₂ … pₙaₙ = q₁b₁ q₂b₂ … qmbm

关键引理：若素数 p | ab，则 p | a 或 p | b

由裴蜀定理（Bézout）证。若 p ∤ a，则 gcd(p, a) = 1（因为 p 是素数，因子只有 1 和 p），由裴蜀有整数 x, y 使 px + ay = 1。两边同乘 b：pbx + aby = b。因为 p | ab，所以 p | aby；又 p | pbx；所以 p | b。✓

主体推导

看 p₁ | N = q₁b₁ q₂b₂ … qmbm。反复用引理（p₁ | XY ⇒ p₁ | X 或 p₁ | Y）：

p₁ | q₁^b₁ · (q₂^b₂ … q_m^b_m)
  ⇒ p₁ | q₁^b₁    或    p₁ | q₂^b₂ … q_m^b_m   ← 引理
继续展开右支
  ⇒ 最终 ∃ k, p₁ | q_k^b_k
因为 p₁ 是素数, 反复用引理
  ⇒ p₁ | q_k        ← 引理（p | q_k · q_k^(b_k-1)）
q_k 也是素数, 只有因子 1 和 q_k
  ⇒ p₁ = q_k        ← 素数的定义

重新排序 q 使 p₁ = q₁。比较 p₁ 的次数：

若 a₁ > b₁:  两边除以 p₁^b₁
  p₁^(a₁-b₁) · p₂^a₂ … p_n^a_n = q₂^b₂ … q_m^b_m   ← 消去
左边仍含因子 p₁ (因 a₁-b₁ ≥ 1)
但右边的 q₂,…,q_m 都 ≠ p₁ (前面已让 q₁=p₁ 排到第一)
⇒ 右边 p₁ 的次数 = 0, 矛盾。

若 a₁ < b₁，对称矛盾。所以 a₁ = b₁。两边消去 p₁a₁，对剩下的项归纳重复，最终得 n = m 且各对 (pᵢ, aᵢ) 与 (qᵢ, bᵢ) 一一对应。∎

把这条定理记住一句话：自然数 → 素数指数序列 是双射。这就是为什么 ⟨a₀, …, aₙ⟩ 能"唯一还原"。

命题 3.27 · 序列数唯一性

PDF 第 4 页

命题

若 ⟨a₀, …, aₙ⟩ = ⟨b₀, …, bm⟩，则 n = m 且对所有 i ≤ n，aᵢ = bᵢ。

白话

序列数是序列的"指纹"——指纹相同必是同一序列。

证明

左边 = ∏ Pᵢaᵢ，右边 = ∏ Pⱼbⱼ。两边是同一个自然数的两种素因子分解。由算数基本定理的唯一性，每个素数（即每个 Pᵢ）在两边的指数必相等。∎

这一句"由算数基本定理即得"是真"显然"——因为序列数本身就是写成素数乘积的形式。

定义 3.28 · ep 函数（解码）

PDF 第 6 页

定义

ep(x, n) ≜ x 的素因子分解中 Pn 的幂次

白话

给一个数 x 和一个下标 n，告诉你"第 n 个素数 Pn 在 x 里的指数是多少"。

具体例子（x = 2² · 3 · 11）

先把 x 补全成"完整素数排列"：

x = 2² · 3¹ · 5⁰ · 7⁰ · 11¹
nPnep(x, n)解读
0222 的指数
1313 的指数
250没出现 → 0
370没出现 → 0
411111 的指数
ep 是序列数 ⟨·⟩ 的"反操作"：⟨·⟩ 把序列藏进素数指数里，ep 把它读出来。

简写 + 命题 3.29 · 机械解码

PDF 第 7 页

约定

ep(x, n) 简写为 epn(x)。下标移到了 ep 上，看着像"取第 n 个元素"。

命题 3.29

epi⟨a₀, …, aₙ⟩ = aᵢ ，i ≤ n

证明（一行）

epi⟨a₀, …, aₙ⟩ = epi( ∏j=0n Pⱼaⱼ ) = "这个数中 Pi 的指数" = aᵢ。∎

至此，⟨·⟩ 和 ep·(·) 构成完美的编码 / 解码对。重要的是这两个操作都是可计算的——给你一个序列数和下标，你就能机械地算出第 i 个元素。

编码 / 解码 小结

补充背景 · 序列数总结，准备进入 Gödel 码
操作符号作用方向
编码⟨a₀, …, aₙ⟩ ≜ ∏ Pᵢaᵢ序列 → 自然数有限序列 ⟶ 一个数
解码epi(x) = x 中 Pᵢ 的指数自然数 → 第 i 个元素一个数 ⟶ 任意位置

这两个操作满足：epi⟨a₀, …, aₙ⟩ = aᵢ（命题 3.29）。

关键性质：编码和解码都能用一个算法机械完成。这就是 Gödel 码"语法可计算"的全部技术基础。

定义 3.30 · 一阶语言的符号集

PDF 第 8 页

回炉：一阶语言由哪些符号构成

(1) 逻辑符（语言无关）

  • V = {xn | n ∈ ℕ}：变元集，无穷可数
  • C = {¬, ∧, ∨, →}：连接词
  • Q = {∀, ∃}：量词
  • E = {≐}：等号（注意是 ≐，不是 ≃ 也不是 =）
  • P = { (, ), ., , }：标点（左括号、右括号、点、逗号）

(2) 非逻辑符（语言相关）

  • ℒf = { fij | i ∈ ℕ, j ∈ Ii }：函数符。i 是元数，j 是序号。当 i = 0 时 f0j 是常元符。
  • ℒP = { Pij | i ∈ ℕ, j ∈ Ji }：谓词符。当 i = 0 时 P0j 是命题符（0 元谓词）。

具体例子

算术语言 𝒜 里的常元 0、1，函数符 +、×，谓词符 < ——在这套形式里写成：

𝒜 中习惯写法形式 (i, j)
0f0,0
1f0,1
+f2,0
×f2,1
<P2,0
符号区分（务必）：
· ≐（U+2250）：一阶语言里的"等词"（语法符号）
· ≃：逻辑等价（如 A ≃ B，第 7 章引入）
· =：元语言里的等于（如 #X = ⟨…⟩）

定义 3.31 · Gödel 码（逻辑符部分）

PDF 第 9 页

记 #X 为 X 的 Gödel 码（X 可以是符号 / 项 / 公式）。逻辑符每个都赋一个序列数：

符号#解读（首位 = 类型标签）
xn⟨0, n⟩0 = 变元
¬⟨1, 0⟩1 = 连接词
∧⟨1, 1⟩
∨⟨1, 2⟩
→⟨1, 3⟩
∀⟨2, 0⟩2 = 量词
∃⟨2, 1⟩
≐⟨3, 0⟩3 = 等号
(⟨4, 0⟩4 = 标点
)⟨4, 1⟩
.⟨4, 2⟩
,⟨4, 3⟩
每个符号都是一个二元组 ⟨类型, 序号⟩ 的序列数。例如 #(¬) = ⟨1, 0⟩ = 2¹ · 3⁰ = 2。
"首位"承担类型识别：解码时一看头位就知道这是变元还是连词还是量词。

Gödel 码（非逻辑符 + 项）

PDF 第 10 页

(2) 非逻辑符

  • #(fij) = ⟨5, i, j⟩ ← 5 = 函数符
  • #(Pij) = ⟨6, i, j⟩ ← 6 = 谓词符
注意这里是三元组（首位 + 元数 + 序号），不是二元组。

(3) 项的 Gödel 码（对项结构归纳）

回炉第 6 章：项要么是变元 / 常元，要么是 f(t₁, …, ti) 形式。

  • (a) 基础：t 为变元或常元时，#t 已被定义（前面已给）。
  • (b) 归纳：t = fij(t₁, …, ti)，定义
    #t = ⟨ #fij, #t₁, …, #ti ⟩
    特别地 f0j 是常元，#f0j 已被定义。

例

设 t = f2,0(x₀, x₁)（即 𝒜 中的 x₀ + x₁）。

#t = ⟨ #f₂,₀,         #x₀,   #x₁  ⟩
    = ⟨ ⟨5, 2, 0⟩,    ⟨0,0⟩, ⟨0,1⟩ ⟩
这就是结构归纳法在"编码"上的应用：项的语法树深下去，Gödel 码就嵌套深下去——但因为每层都是序列数，整体仍是单一自然数。

Gödel 码 · 首位对照速查表

补充背景

把"首位 → 符号类型"的对照集中起来，方便看到一个 Gödel 码立刻判类型。

首位类型结构例子
0变元⟨0, n⟩xn
1连接词⟨1, k⟩ 或递归 ⟨1, …, #A, #B⟩¬, ∧, ∨, →
2量词 / 量化公式⟨2, k⟩ 或 ⟨2, …, #x, #., #A⟩∀, ∃
3等号 / 等式⟨3, 0⟩ 或 ⟨3, …, #t, #s⟩≐
4标点⟨4, k⟩( ) . ,
5函数符 / 项⟨5, i, j⟩ 或 ⟨5, …, #t₁, …⟩fij(…)
6谓词符 / 原子公式⟨6, i, j⟩ 或 ⟨6, …, #t₁, …⟩Pij(…)
关键 idea：首位 = 类型 tag。看到 #X 后用 ep₀(#X) 就能机械地判别 X 是什么种类。

定义 3.31 (4) · 公式的 Gödel 码

PDF 第 11 页

同样对公式结构归纳定义 #A：

  • (a) 等式：#(t ≐ s) = ⟨ #(≐), #t, #s ⟩
  • (b) 谓词公式：#(Pij(t₁, …, ti)) = ⟨ #Pij, #t₁, …, #ti ⟩。特别地命题符 #(P0j) 已被定义。
  • (c) 连接词复合：
    #(¬A) = ⟨ #(¬), #A ⟩
    #(A * B) = ⟨ #(*), #A, #B ⟩，其中 * ∈ {∧, ∨, →}
  • (d) 量化公式：
    #(∀x . A) = ⟨ #(∀), #x, #(.), #A ⟩
    #(∃x . A) = ⟨ #(∃), #x, #(.), #A ⟩
"."在量化公式语法里是分隔符（∀x . A 而不是 ∀x A），所以编码里要带上 #(.) = ⟨4, 2⟩。
⚠ 这是第 6 章「结构唯一性定理」的算术化版本：原本说"公式从子公式唯一构造"，现在变成"Gödel 码从子码唯一确定的嵌套序列数"。两者本质同一件事。

例 · #(∀x₀ P2,6(x₀, x₁))

PDF 第 12 页

逐层展开

#(∀x₀ P₂,₆(x₀, x₁))
= ⟨ #(∀),  #(x₀),  #(P₂,₆(x₀, x₁)) ⟩           ← 量化公式规则 (d)，省略了 #(.)
= ⟨ ⟨2,0⟩, ⟨0,0⟩, ⟨ #(P₂,₆),  #(x₀), #(x₁) ⟩ ⟩  ← 谓词公式规则 (b)
= ⟨ ⟨2,0⟩, ⟨0,0⟩, ⟨ ⟨6, 2, 6⟩, ⟨0,0⟩, ⟨0,1⟩ ⟩ ⟩ ← 非逻辑符 #(P_ij) = ⟨6, i, j⟩
最里层 ⟨6,2,6⟩, ⟨0,0⟩, ⟨0,1⟩ 各自展开成具体数:
  ⟨6,2,6⟩ = 2⁶ · 3² · 5⁶ = 9 000 000
  ⟨0,0⟩  = 2⁰ · 3⁰ = 1
  ⟨0,1⟩  = 2⁰ · 3¹ = 3
所以最里层 = ⟨9000000, 1, 3⟩ = 2^9000000 · 3¹ · 5³
最外层 = 2⁴ · 3 · 5^(2^9000000 · 3 · 5³)              ← 怪兽数
⚠ 这个数大到天文级别（2 的 9 百万次方……再做指数）。但它仍然是一个良定义的自然数；不需要算它，只需要知道它存在且唯一。
"数字大"不重要，重要的是"从这个数能机械还原出公式"——这就是下一页定理 3.32 要总结的。

定理 3.32 · 唯一对应

PDF 第 13 页

ℒ 中的符号、项和公式对应唯一的一个自然数（其 Gödel 码），且存在一个算法从 Gödel 码机械还原原 ℒ 语法对象。

白话

语法对象 ↔ 自然数 是双射，且可计算可逆。

为什么

  • 唯一性：每个序列数 ⟨…⟩ 由素数指数唯一确定（命题 3.27）。整个嵌套结构是序列数套序列数，每层都唯一。
  • 可计算还原：用 ep₀(·) 取首位 → 知道是哪类符号 → 用 ep₁(·), ep₂(·) … 取出子部分 → 递归。终止条件是首位 0/1/2/3/4 的原子符号。
这一条定理是"语法是算术的"的精确说法。它使得"哪些公式可证 / 不可证"这类元问题，原则上可以用算术工具研究——这是 Gödel 不完备性定理的入口。
本章用到这一条的"双射 + 可计算"事实，至于完整算法的伪代码，下一讲（讲可计算性时）才会展开。

Hintikka 集 · 动机（重要！）

补充背景 · 必看，否则后面 17 条定义看似随意

接下来要登场的"Hintikka 集"是一个公式集合 Ψ，它的定义看起来非常长（17 条规则），但动机非常简单：

Hintikka 集 = 一个公式集合，里面的复合公式都已经"分解到底"了。

什么叫"分解到底"

意思是：如果一个复合公式 X 在 Ψ 里，那么它的"语义构件"也必须在 Ψ 里。例如：

  • A ∧ B ∈ Ψ ⇒ A ∈ Ψ 且 B ∈ Ψ ←（分解成两个证人）
  • A ∨ B ∈ Ψ ⇒ A ∈ Ψ 或 B ∈ Ψ ←（至少一个证人）
  • ∀x. A ∈ Ψ ⇒ 对所有项 t，A[t/x] ∈ Ψ ←（穷举所有实例）
  • ∃x. A ∈ Ψ ⇒ 存在某项 t，A[t/x] ∈ Ψ ←（至少一个见证）

再加上一致性的最弱要求："原子公式 A 和 ¬A 不能都在里面"（规则 1）。

为什么这种集合能造出模型

因为既然每个复合公式都已经被拆到原子级别，那我们只要让模型中"原子公式 ∈ Ψ"的为真、否则的为假，整个 Ψ 就自动逐级为真——这正是引理 3.41 要证的。

这一类"向下饱和的语法封闭集"也叫 downward saturated set。Hintikka 集 = 一阶版本的 downward saturated set + 等词规则。

定义 3.33 · Hintikka 集（完整 17 条）

PDF 第 17 页 · 含全部 17 条

设 ℒ 为一阶语言，Ψ 为 ℒ 的公式集，令 T 为全体 ℒ 项的集合。Ψ 是 Hintikka 集 指：

① 原子一致（必须）

1. 若公式 A 为原子公式，则 A 和 ¬A 不能都 ∈ Ψ。（最弱一致性）

② α 型 · 双支都 ∈ Ψ

2. 若 ¬¬A ∈ Ψ，则 A ∈ Ψ。（双否消去）

4. 若 ¬(A → B) ∈ Ψ，则 A ∈ Ψ 且 ¬B ∈ Ψ。

5. 若 A ∧ B ∈ Ψ，则 A ∈ Ψ 且 B ∈ Ψ。

8. 若 ¬(A ∨ B) ∈ Ψ，则 ¬A ∈ Ψ 且 ¬B ∈ Ψ。

③ β 型 · 至少一支 ∈ Ψ

3. 若 A → B ∈ Ψ，则 ¬A ∈ Ψ 或 B ∈ Ψ。

6. 若 ¬(A ∧ B) ∈ Ψ，则 ¬A ∈ Ψ 或 ¬B ∈ Ψ。

7. 若 A ∨ B ∈ Ψ，则 A ∈ Ψ 或 B ∈ Ψ。

④ γ 型 · 对全体项实例化

9. 若 ∀x. A ∈ Ψ，则 ∀t ∈ T，A[t/x] ∈ Ψ。

12. 若 ¬∃x. A ∈ Ψ，则 ∀t ∈ T，¬A[t/x] ∈ Ψ。

⑤ δ 型 · 存在见证项

10. 若 ¬∀x. A ∈ Ψ，则 ∃t ∈ T，¬A[t/x] ∈ Ψ。

11. 若 ∃x. A ∈ Ψ，则 ∃t ∈ T，A[t/x] ∈ Ψ。

⑥ ε 型 · 等词公理

13. t ≐ t ∈ Ψ （自反）

14. t ≐ s → s ≐ t ∈ Ψ （对称）

15. t ≐ s → (s ≐ u → t ≐ u) ∈ Ψ （传递）

16. 若 f 为 n 元函数，(⋀ᵢ tᵢ ≐ sᵢ) → f(t₁,…,tₙ) ≐ f(s₁,…,sₙ) ∈ Ψ （函数同应于相等参数 = 相等结果）

17. 若 P 为 n 元谓词，(⋀ᵢ tᵢ ≐ sᵢ) → (P(t₁,…,tₙ) → P(s₁,…,sₙ)) ∈ Ψ （等参数下谓词同真）

⚠ 区分 γ 和 δ：
· 9, 12 要求"对所有 t ∈ T"
· 10, 11 要求"存在 t ∈ T"
这是后面引理 3.41 ∀ / ∃ 两条情况的分水岭。
17 条规则可以看作一张"语法层面的检查表"：把任何复合公式扔进去都能机械地继续拆下去。

定理

若 Ψ 为 Hintikka 集，则 Ψ 可满足（即存在结构和赋值使 Ψ 中每个公式都为真）。

这条定理为什么是本章核心

"语法上看起来自洽"（Hintikka 性质）⟹ "语义上真的有模型"。这就是把语法对应到语义的关键桥梁。

证明思路（接下来 p19-p31 都在做这件事）

  1. p19 · 在项集 T 上定义等价关系 ~（s ~ t ⇔ s ≐ t ∈ Ψ）。
  2. p20-p23 · 证明 ~ 对函数 / 谓词有"良行为"（引理 3.38，congruence）。
  3. p25 · 用项的等价类 [t] 作为论域，定义模型 ℍ。
  4. p26-p31 · 对公式结构归纳，证 ℍ ⊨_σ Ψ。
这种"用语法对象本身造模型"的构造叫 项模型 / Henkin 模型。它的精妙在于：模型完全由 Ψ 决定，所以 Ψ 当然在它上为真。

定义 3.35-3.37 · 项上的等价关系

定义 3.35 · 二元关系 ~

s ~ t 指 s ≐ t ∈ Ψ

白话：在项集 T 上定义"我们认为相等"的关系——若 Ψ 里写明 "s ≐ t"，那这两个项就被认作等价。

命题 3.36

~ 是等价关系（自反 / 对称 / 传递）。证明留作习题——但你可以一眼看到这正是 Hintikka 规则 13-15 兜住的：

  • 自反：t ≐ t ∈ Ψ（规则 13）→ t ~ t ✓
  • 对称：t ≐ s → s ≐ t ∈ Ψ（规则 14）→ 若 s ≐ t ∈ Ψ 则 s ≐ t → t ≐ s 用规则 3（β）展开得 t ≐ s ∈ Ψ。（需要小心规则 3 的"或"分支：若 ¬(s≐t)∈Ψ 则与前提矛盾，所以只能取右支）
  • 传递：类似用规则 15。

定义 3.37 · 等价类 [t]

设 t ∈ T。令 [t] = { s ∈ T | s ~ t } 为 t 关于 ~ 的等价类。则 [s] = [t] ⇔ s ~ t。

这一步是 Henkin 项模型构造的关键技巧：把 Ψ 中"被认为相等的项"打包成一个对象。下一步定义模型时，论域 = 全部等价类的集合。

为什么要等价类

补充背景 · 等价类不是技巧，是必需

来用算术语言 𝒜 举一个具体例子：

假设 Ψ 里包含 "2 + 2 ≐ 4"（在 𝒜 中是常元 + 函数符的项）。
"2+2" 和 "4" 在 ℒ 里是两个不同的项（语法形式不一样）。

如果直接拿"项集 T 当论域"会怎样

  • 论域里会同时有 "2+2" 和 "4" 两个不同对象
  • 但 Ψ 要求 "2+2 ≐ 4" 为真
  • 而等词 ≐ 的语义恰恰要求两边解释为同一个对象
  • 矛盾 → 模型造不出来

解决

把"被 Ψ 声明相等"的项合并成一个等价类 [t]。论域用 H = {[t] | t ∈ T}。这样：

  • [2+2] 和 [4] 是同一个对象（因为 2+2 ~ 4）
  • 等词的语义 [s] = [t] 自动满足
这就是为什么要定义 ~，再用 [t] 当论域元素：让"语法相等" ≅ "语义相等"。
⚠ 这种把项打包成类的做法是 商结构 / quotient construction。要让它良定义，必须证明所有函数 / 谓词都"尊重" ~ —— 这就是下一页引理 3.38。

引理 3.38 · ~ 对函数 / 谓词是 congruence

PDF 第 23 页 · 含完整证明 + 规则标注

引理

设 [tᵢ] = [sᵢ]（i = 1, …, n）。则：

  1. 任何 n 元函数 f，[f(t₁, …, tₙ)] = [f(s₁, …, sₙ)]。
  2. 任何 n 元谓词 P，若 P(t₁, …, tₙ) ∈ Ψ，则 P(s₁, …, sₙ) ∈ Ψ。

白话翻译

  • (1) "相等的输入给函数 → 输出也相等"。这让 fH 对等价类作用是良定义的。
  • (2) "相等的输入下谓词同真同假"。这让 PH 对等价类作用是良定义的。

证明 · (1)

要证 [f(t)] = [f(s)]，即 f(t₁,…,tₙ) ≐ f(s₁,…,sₙ) ∈ Ψ。

由 [tᵢ] = [sᵢ]
  ⇒ tᵢ ~ sᵢ                         ← 等价类的定义 3.37
  ⇒ tᵢ ≐ sᵢ ∈ Ψ                     ← ~ 的定义 3.35
  ⇒ ¬(tᵢ ≐ sᵢ) ∉ Ψ                  ← Hintikka 规则 1（原子一致）

又 Hintikka 规则 16:
  (⋀ᵢ tᵢ ≐ sᵢ) → f(t₁,…,tₙ) ≐ f(s₁,…,sₙ) ∈ Ψ

把这个蕴含式记为 X → Y, 其中
  X = ⋀ᵢ tᵢ ≐ sᵢ
  Y = f(t₁,…,tₙ) ≐ f(s₁,…,sₙ)

应用 Hintikka 规则 3 (A → B ∈ Ψ ⇒ ¬A ∈ Ψ 或 B ∈ Ψ):
  ¬X ∈ Ψ  或  Y ∈ Ψ
但 ¬X = ¬⋀ᵢ tᵢ ≐ sᵢ; 若它 ∈ Ψ, 用规则 6 (¬∧) 反复展开
  ⇒ 某个 ¬(tᵢ ≐ sᵢ) ∈ Ψ              ← 与上面 ¬(tᵢ ≐ sᵢ) ∉ Ψ 矛盾
所以必为右支:
  Y = f(t₁,…,tₙ) ≐ f(s₁,…,sₙ) ∈ Ψ   ← (1) 得证 ∎

证明 · (2)

"与 1 同理"：用 Hintikka 规则 17（谓词相容）代替规则 16，其余推理结构相同。

由 [tᵢ] = [sᵢ] ⇒ tᵢ ≐ sᵢ ∈ Ψ ⇒ ¬(tᵢ ≐ sᵢ) ∉ Ψ   ← 规则 1
Hintikka 规则 17:
  (⋀ᵢ tᵢ ≐ sᵢ) → (P(t₁,…,tₙ) → P(s₁,…,sₙ)) ∈ Ψ
用规则 3 拆外层蕴含 → 右支必成立:
  P(t₁,…,tₙ) → P(s₁,…,sₙ) ∈ Ψ
再用规则 3 拆这个蕴含 → ¬P(t₁,…,tₙ) ∈ Ψ 或 P(s₁,…,sₙ) ∈ Ψ
但前提 P(t₁,…,tₙ) ∈ Ψ ⇒ ¬P(t₁,…,tₙ) ∉ Ψ           ← 规则 1
所以 P(s₁,…,sₙ) ∈ Ψ                                ← (2) 得证 ∎
这就是 Henkin 项模型能成立的"良定义性"基石：Hintikka 规则 16, 17 + 规则 1 一起保证了"商结构是合法的"。

(a) 由 s ≐ t ∈ Ψ
  ⇒ s ~ t                              ← ~ 的定义 (定义 3.35)
  ⇒ [s] = [t]                          ← 等价类的定义 (定义 3.37)
  ⇒ s_H[σ] = t_H[σ]                    ← 引理 3.40 (t_H[σ] = [t])
  ⇒ (s ≐ t)_H[σ] = T                   ← 等词的语义 (第 7 章)
  得 (a) 成立。

(b) 由 ¬(s ≐ t) ∈ Ψ
  ⇒ s ≐ t ∉ Ψ                          ← Hintikka 规则 1 (原子一致)
  ⇒ [s] ≠ [t]                          ← ~ 和等价类定义 (反向)
  ⇒ s_H[σ] ≠ t_H[σ]                    ← 引理 3.40
  ⇒ (¬(s ≐ t))_H[σ] = T                ← 等词语义 + ¬ 语义
  即 (s ≐ t)_H[σ] = F, 得 (b) 成立。

情况 ¬ · A = ¬B

(a) A ∈ Ψ
  ⇒ ¬B ∈ Ψ
  ⇒ B_H[σ] = F                         ← 对子公式 B 的归纳假设的 (b)
  ⇒ A_H[σ] = (¬B)_H[σ] = T             ← ¬ 的语义

(b) ¬A ∈ Ψ
  ⇒ ¬¬B ∈ Ψ
  ⇒ B ∈ Ψ                              ← Hintikka 规则 2 (双否消去)
  ⇒ B_H[σ] = T                         ← 对子公式 B 的归纳假设的 (a)
  ⇒ A_H[σ] = (¬B)_H[σ] = F             ← ¬ 的语义
连接词情况都是这种模式：用 Hintikka 规则把外层结构剥掉，落到子公式上 → 用归纳假设。

情况 ∧ · A = B ∧ C

(a) A ∈ Ψ
  ⇒ B ∧ C ∈ Ψ
  ⇒ B ∈ Ψ 且 C ∈ Ψ                     ← Hintikka 规则 5 (α 型)
  ⇒ B_H[σ] = C_H[σ] = T                ← 对 B, C 的归纳假设 (a)
  ⇒ (B ∧ C)_H[σ] = T                   ← ∧ 的语义
  ⇒ A_H[σ] = T。

(b) ¬A ∈ Ψ
  ⇒ ¬(B ∧ C) ∈ Ψ
  ⇒ ¬B ∈ Ψ 或 ¬C ∈ Ψ                   ← Hintikka 规则 6 (β 型)
  ⇒ B_H[σ] = F 或 C_H[σ] = F           ← 对 B, C 的归纳假设 (b)
  ⇒ (B ∧ C)_H[σ] = F                   ← ∧ 的语义
  ⇒ A_H[σ] = F。

情况 ∀ · A = ∀x. B

这是整个证明里最精妙的一步——要让"语法上的对所有项 t"对接到"语义上的对所有论域元素 u"。看每一行用了哪条工具：

(a) ∀x. B ∈ Ψ
  ⇒ B[t/x] ∈ Ψ, 对 ∀t ∈ T                            ← Hintikka 规则 9 (γ)
  ⇒ B[t/x]_H[σ] = T, 对 ∀t ∈ T                       ← 对 B[t/x] 的归纳假设 (a)
  ⇒ B_H[σ[x := t_H[σ]]] = T, 对 ∀t ∈ T               ← 替换引理 (第 7 章)
  ⇒ B_H[σ[x := [t]]] = T, 对 ∀t ∈ T                  ← 引理 3.40 (t_H[σ] = [t])
  ⇒ B_H[σ[x := u]] = T, 对 ∀u ∈ H                    ← H = {[t] | t ∈ T}, 即遍历 t 就是遍历 u
  ⇒ (∀x. B)_H[σ] = T                                  ← ∀ 的语义定义 (第 7 章)
  ⇒ A_H[σ] = T。

每一步的关键

  • ① Hintikka 规则 9：把 ∀x. B 实例化到 Ψ 中"所有项"上 — 这是"语法的全称展开"。
  • ② 归纳假设：B[t/x] 是比 ∀x. B 简单的公式（结构归纳的有效性来自第 6 章），所以可以用归纳假设把"∈ Ψ"转成"真"。
  • ③ 替换引理（第 7 章学过）：B[t/x]_H[σ] = B_H[σ[x := t_H[σ]]]。
    白话：用项 t 替换变元 x，等价于先解释 t、再把这个语义对象赋给 x。
  • ④ 引理 3.40：t_H[σ] = [t]。把"项的语义" 改写成"项的等价类"。
  • ⑤ 关键观察：因为 H = {[t] | t ∈ T}，遍历所有项 t 给出的 [t] 恰好就是整个论域 H。所以"对所有 t" 等于"对所有 u ∈ H"。
  • ⑥ ∀ 的语义：对所有论域元素都真 ⇒ ∀x. B 在 σ 下为真。
⚠ 第 ⑤ 步是项模型的精髓。在一般结构里，"对所有项 t" 和"对所有论域元素 u" 是两个不同的概念（论域元素可能没有项命名它）。但在项模型 ℍ 里两者恰好一致——这就是为什么 Hintikka 规则 9 的"对所有 t" 足以兑现 ∀ 的语义。
