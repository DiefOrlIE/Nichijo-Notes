1 · 数据表示与预处理

这是《3.1》的前半部分。所有模型的输入都是数字向量，理解这一层，后面 PCA 的"中心化""规范化"才不会突兀。

1.1 把现实问题变成向量

机器学习处理的输入永远是一个数值向量 $\mathbf{x}\in\mathbb{R}^D$。比如房价预测里，一套房子被表示成 $\mathbf{x}=(\text{面积},\ \text{卧室数},\ \text{楼层},\dots)$，每一个分量叫一个特征（feature），$D$ 就是特征维度。

$N$ 个样本叠起来就是一个矩阵。本次实验里约定：

数据矩阵 $X$ 形状为 $(N, D)$ —— 一行一个样本，一列一个特征。 $$X=\begin{pmatrix} x_{11} & x_{12} & \cdots & x_{1D}\\ x_{21} & x_{22} & \cdots & x_{2D}\\ \vdots & & & \vdots\\ x_{N1} & x_{N2} & \cdots & x_{ND}\end{pmatrix}$$
注意一个表示差异（PCA 会用到）： 讲义《3.1》里为了推导方便，把 $X$ 写成列是样本（$D\times N$），所以协方差写成 $XX^\mathsf{T}$。而本题的 $X$ 是行是样本（$N\times D$），对应的协方差就要写成 $X^\mathsf{T}X$。方向别记反——判断标准很简单：协方差矩阵一定是 $D\times D$ 的（特征 × 特征）。

特征分两类：

  • 数值型（numerical）：重量、长度、面积，本身就是数，可直接计算。
  • 类别型（categorical）：颜色=红/黄/蓝，大小=大/中/小。不能直接当数算，要先编码（如 one-hot，或有序类别映射成 $1,0,-1$）。
    三道题用的数据都是数值型，类别型了解即可。

1.2 规范化（Normalization）

不同特征量纲差别可能很大（面积上千、卧室数个位数）。如果直接算距离或方差，大数值的特征会压倒小数值的特征。所以常要先把各特征拉到可比的范围。两种常见做法：

方法公式效果
min-max $f_i'=\dfrac{f_i-f_{\min}}{f_{\max}-f_{\min}}$ 线性缩放到 $[0,1]$
z-score $f_i'=\dfrac{f_i-\bar f}{\sigma}$ 变成均值 0、标准差 1
和这三道题的关系： PCA 的第一步"中心化"（减均值）其实就是 z-score 的前半段；KNN 算距离前理论上也该规范化，否则量纲大的特征主导距离。本次三道题的判题数据都没要求你额外规范化（题一只要求中心化），所以按题目说的做就行，但要知道"为什么实际工程里要规范化"。

2 · 降维与主成分分析 PCA

对应《3.1》后半。这是题一的全部理论。

2.1 为什么要降维

当维度 $D$ 很高时（课程成绩几十门、文本几千个词、图像几万像素），会带来：难以可视化、容易受噪声干扰、计算开销大、模型容易过拟合。降维就是用更少的维度 $d\ (d

2.2 核心直觉：方差越大，信息越多

PCA 是无监督的（只看 $X$，不看标签 $y$）。它的核心判断是一句话：

数据在哪个方向上"摊得越开"（方差越大），那个方向就越重要。

想象一团沿斜线分布的点云。如果硬投影到水平轴，很多点会挤在一起、难以区分；但如果投影到点云拉伸最长的那个方向，点与点之间区分度最高，信息损失最小。PCA 要找的就是这个"最大方差方向"，称为第一主成分（1st PC）。找完第一个，再在与它正交的方向里找方差第二大的，是第二主成分，依此类推。

怎么量化"投影到方向 $\mathbf{v}$ 后的方差"？设 $\mathbf{v}$ 是单位向量（$\|\mathbf{v}\|=1$），样本 $\mathbf{x}_i$ 在 $\mathbf{v}$ 上的投影是标量 $\mathbf{v}^\mathsf{T}\mathbf{x}_i$。先把数据中心化（减去均值，记中心化后仍写 $\mathbf{x}_i$），则投影后的方差是：

$$\text{Var}=\frac1N\sum_{i=1}^N (\mathbf{v}^\mathsf{T}\mathbf{x}_i)^2$$

PCA 的优化目标就是：在 $\|\mathbf{v}\|=1$ 的约束下，最大化这个方差。

为什么最后落到"特征值分解"

把上式整理成矩阵形式（这里 $X$ 行是样本，所以用 $X^\mathsf{T}X$）：

$$\frac1N\sum_i (\mathbf{v}^\mathsf{T}\mathbf{x}_i)^2 \;=\; \mathbf{v}^\mathsf{T}\Big(\tfrac1N X^\mathsf{T}X\Big)\mathbf{v} \;=\;\mathbf{v}^\mathsf{T} C\, \mathbf{v}$$

其中 $C=\frac1N X^\mathsf{T}X$ 正是中心化数据的协方差矩阵（$D\times D$）。带约束求极值（拉格朗日乘子法），结论非常干净：

$$C\mathbf{v}=\lambda \mathbf{v}$$ 也就是说，最优的投影方向 $\mathbf{v}$ 恰好是协方差矩阵 $C$ 的特征向量，对应的特征值 $\lambda$ 就等于投影后的方差。

所以：特征值越大 → 那个方向方差越大 → 越重要。把特征值从大到小排，取前 $d$ 个特征向量，就是我们要的 $d$ 个主成分。又因为 $C$ 是对称矩阵，它的特征向量天然两两正交，正好满足"各主成分互相正交"的要求。

关于 $\frac1N$： 协方差是除以 $N$ 还是 $N-1$、甚至不除，都不影响特征向量方向（只是把所有特征值同比例缩放），也不影响方差贡献率（比值约掉了）。所以题一里你用 np.cov 还是手写 X.T@X 都能得到同样的主成分。

2.3 PCA 完整流程（题一就是这五步）

  1. 中心化：对每一列（特征）减去该列均值，得到 $X_c$。$\bar{\mathbf{x}}=\frac1N\sum_i \mathbf{x}_i$，$X_c = X-\bar{\mathbf{x}}$。
  2. 算协方差矩阵：$C=\frac1N X_c^\mathsf{T}X_c$，形状 $D\times D$。
  3. 特征值分解：求 $C$ 的特征值 $\lambda_1,\dots,\lambda_D$ 和对应特征向量。对称矩阵用 np.linalg.eigh 更稳。
  4. 排序选前 d 个：按特征值从大到小排序，取前 $d$ 个特征值对应的特征向量，拼成投影矩阵 $W$（$D\times d$）。
  5. 投影：$Z = X_c\, W$，形状 $(N,d)$，就是降维结果。
最大的坑：eigh 返回的特征值是从小到大的！ NumPy 的 np.linalg.eigh 默认升序返回。你必须自己反转顺序，把最大的排到前面，否则取的是方差最小的方向，全错。

2.4 方差贡献率（题目要输出的第一行）

第 $k$ 个主成分的方差贡献率 = 它的特征值占所有特征值之和的比例：

$$r_k=\frac{\lambda_k}{\sum_{j=1}^{D}\lambda_j}$$

直觉：它衡量"这个主成分保留了原始数据百分之多少的信息（方差）"。前 $d$ 个加起来越接近 1，说明降维丢的信息越少。验证示例：题目数据两个特征值是 $8$ 和 $2$，第一主成分贡献率 $=8/(8+2)=0.8$，正好对上题目输出的 0.8000。

2.5 特征向量符号唯一化（题目的特殊规定）

特征向量有个天然的不确定性：$\mathbf{v}$ 和 $-\mathbf{v}$ 都是合法特征向量（方向相反，但都代表同一条轴）。不同库、不同机器算出来的正负号可能不同，会导致投影结果差一个负号。为了让所有人输出一致，题目强制规定：

对每个特征向量，找到它绝对值最大的那个分量；如果该分量是负数，就把整个向量乘以 −1。

这样保证"最大幅度的分量永远为正"，符号就唯一了。落实到代码就是几行（见下方解题）。

★ 题一 · 解题思路与关键片段

输入首行 N D d，接着 $N$ 行每行 $D$ 个实数。输出第一行 $d$ 个方差贡献率（4 位小数），再输出 $N$ 行降维后矩阵（每行 $d$ 个数，4 位小数）。

把 2.3 的五步直接翻译成代码骨架，关键处给提示、留白：

import numpy as np

def pca_reduce(X, d):
    # X: (N, D)。下面每一步对应讲稿 2.3 的五步

    # 1. 中心化：按列(axis=0)求均值再减。keepdims 让广播更稳
    mean = X.mean(axis=0)
    Xc = X - mean

    # 2. 协方差矩阵 (D,D)。行是样本所以是 Xc.T @ Xc
    C = (Xc.T @ Xc) / X.shape[0]      # 除不除 N 都不影响结果

    # 3. 对称矩阵特征分解；eigh 返回的特征值是【升序】！
    eigvals, eigvecs = np.linalg.eigh(C)   # eigvecs 的每一【列】是一个特征向量

    # 4. 反转成降序，取前 d 个
    order = np.argsort(eigvals)[::-1]      # 从大到小的下标
    # TODO: 用 order 重排 eigvals 和 eigvecs(按列重排)，再切前 d 个
    # top_vals = ...   top_vecs = ...   (top_vecs 形状应为 (D, d))

    # 5. 符号唯一化：对每个特征向量(每一列)处理 —— 讲稿 2.5
    for j in range(top_vecs.shape[1]):
        col = top_vecs[:, j]
        k = np.argmax(np.abs(col))         # 绝对值最大分量的下标
        if col[k] < 0:
            top_vecs[:, j] = -col          # 整列翻转

    # 方差贡献率 = 该特征值 / 全部特征值之和（分母用所有 D 个！）
    ratios = top_vals / eigvals.sum()

    # 6. 投影：Z = Xc @ W
    Z = Xc @ top_vecs                      # (N, d)
    return ratios, Z
题一三个最易错点：
  1. 没反转 eigh 的升序 → 取到了最小方差方向。
  2. 方差贡献率分母用错 → 必须是所有 $D$ 个特征值之和，不是前 $d$ 个之和。
  3. 忘了符号唯一化 → 投影值正负号和标准答案相反。
输出格式：题目要求 4 位小数，打印时用 f"{x:.4f}"。注意 -0.0000 这种边界——一般判题能接受，若不放心可在格式化前加 x + 0.0 或对极小值做归零处理。

3 · 有监督学习总览

对应《3.2》开头到评估部分。题二、题三都属于有监督学习，先建立共同的框架。

3.1 什么是有监督学习 · 回归 vs 分类

有监督学习 = 用一堆带答案的样本 $(\mathbf{x}_i, y_i)$ 去学一个函数 $f$，使 $f(\mathbf{x})$ 尽量接近真实 $y$。$\mathbf{x}$ 是输入，$y$ 是标签（答案）。按 $y$ 的类型分两大类：

回归 Regression分类 Classification
输出 $y$连续实数（如房价 369.0）离散类别（如 0/1，或多类）
例子预测房价判断西瓜熟没熟、邮件是否垃圾
本次题目—题二 KNN、题三 逻辑斯蒂回归都是分类
易混点："逻辑斯蒂回归"名字里带"回归"，但它做的是分类。名字的由来是它在内部对"对数几率"做了线性回归——后面 5.2 会讲。

3.2 怎么评估好坏 · 过拟合

训练完模型要看它表现如何。评估指标按任务分：

  • 回归：均方误差 MSE $=\frac1m\sum_i (f(\mathbf{x}_i)-y_i)^2$，越小越好。
  • 分类：错误率 $E=\frac1m\sum_i \mathbb{1}[f(\mathbf{x}_i)\neq y_i]$；精度 $\text{acc}=1-E$，越大越好。（$\mathbb{1}[\cdot]$ 是指示函数，括号内成立取 1 否则 0）

更重要的是在哪批数据上评估：

训练误差（经验误差）：在见过的训练集上的误差。
泛化误差：在没见过的测试集上的误差 —— 这才是我们真正关心的。
为此要把数据划分成训练集 / 测试集（两者不能重叠），常用留出法、交叉验证。

过拟合 vs 欠拟合：模型太简单 → 训练集都学不好（欠拟合）；模型太复杂 → 把训练集的噪声都背下来了，训练误差极低但泛化差（过拟合）。如讲义"识别猫"的例子：把训练集的猫记得死死的，遇到一只没见过姿势的猫反而认不出。降维（PCA）有时能降噪、缓解过拟合——这就把第一部分和这里连起来了。

4 · K 近邻 KNN

对应《3.2》KNN 部分。题二的全部理论。

4.1 核心思想：近朱者赤

一个样本的类别，由离它最近的 $K$ 个训练样本投票决定。

这背后的假设是"相近的样本，标签也相近"。预测一个新样本时：算它和所有训练样本的距离 → 找最近的 $K$ 个 → 这 $K$ 个里哪个类别票数最多，就预测成哪个类。

惰性学习（Lazy Learning）： KNN 没有训练过程，不学任何参数，只是把训练集存起来。所有计算都推迟到预测时才做。代价是：预测慢（每次都要和全部训练样本比），且要存下整个训练集。

4.2 距离怎么算

对两个 $D$ 维样本 $\mathbf{x},\mathbf{y}$，题目要支持两种距离：

类型 T名称公式直觉
1欧氏距离$\sqrt{\sum_{i=1}^D (x_i-y_i)^2}$两点间直线距离
2曼哈顿距离$\sum_{i=1}^D |x_i-y_i|$沿网格走的"出租车"距离

这两者其实是闵可夫斯基距离 $L_p=\big(\sum_i|x_i-y_i|^p\big)^{1/p}$ 的特例：$p=1$ 是曼哈顿，$p=2$ 是欧氏。了解即可，题目只要这两种。

小优化（不影响结果）： 比较欧氏距离的大小时，开不开根号排序结果一样（根号是单调的）。所以只为了排序找近邻，可以省掉 sqrt。不过为了代码清晰、避免出错，按公式老实写也完全没问题。

4.3 投票与两条平票规则

题目对"排序"和"投票"都规定了确定性的 tie-break 规则，必须严格照做，否则边界样本会判错：

  1. 排序时距离相同： 训练集中先出现的样本排在前面。→ 这要求排序是稳定的，并且把"原始下标"作为次级比较键。
  2. 投票时多个类别票数相同： 选类别编号最小的那个类。

4.4 K 的选择（理解，不用调参）

$K$ 小（如 1）：决策边界贴合数据细节，但容易被噪声/异常点带偏。$K$ 大：边界更平滑、更稳，但可能模糊掉真实结构；$K$ 等于全体训练样本时，无论输入是什么都预测成"训练集里最多的那个类"。本题 $K$ 由输入给定，理解这个权衡即可。

★ 题二 · 解题思路与关键片段

输入首行 N D Q K T；接着 $N$ 行训练样本（$D$ 个实数 + 1 个整数标签）；再 $Q$ 行测试样本。输出$Q$ 行预测类别。

对每个测试样本，做"算距离 → 带下标排序 → 取前 K → 计票 → 选最小编号"。关键片段：

def distance(a, b, T):
    if T == 1:                                  # 欧氏
        return sum((ai-bi)**2 for ai,bi in zip(a,b)) ** 0.5
    else:                                     # 曼哈顿 (T==2)
        return sum(abs(ai-bi) for ai,bi in zip(a,b))

def knn_predict(train_X, train_y, test_X, K, T):
    preds = []
    for x in test_X:
        # 1. 算到每个训练样本的距离，连同【原始下标 i】一起存
        items = [(distance(x, train_X[i], T), i) for i in range(len(train_X))]

        # 2. 排序：先按距离升序，距离相同按下标 i 升序(=先出现的在前)
        items.sort(key=lambda t: (t[0], t[1]))

        # 3. 取前 K 个邻居的标签
        neighbor_labels = [train_y[i] for _, i in items[:K]]

        # 4. 计票
        votes = {}
        for lab in neighbor_labels:
            votes[lab] = votes.get(lab, 0) + 1

        # 5. 选票数最多者；平票选【类别编号最小】
        best = max(votes.keys(), key=lambda c: (votes[c], -c))
        # 解释:(票数大优先, 票数相同则 -c 大即 c 小优先)。TODO:想清楚为什么是 -c
        preds.append(best)
    return preds
题二易错点：
  1. 平票选最小编号：用 max(..., key=(票数, -类别)) 这个技巧可以一行搞定；自己写 if 比较也行，但别忘了这条规则。
  2. 距离相同的稳定排序：把原始下标 i 作为第二排序键，天然满足"先出现的在前"。
  3. T 的分支：1 是欧氏、2 是曼哈顿，别记反。

5 · 逻辑斯蒂回归 Logistic Regression

对应《3.2》分类模型部分。题三的全部理论。这是三道题里数学链条最长的一道，跟着直觉走就不难。

5.1 从线性回归到分类

线性回归能算出一个实数 $z=\mathbf{w}^\mathsf{T}\mathbf{x}+b$。但分类要的是 0/1，怎么从实数 $z$ 跳到类别？最朴素的想法是用阶跃函数：$z>0$ 判 1，$z<0$ 判 0。问题是阶跃函数在 0 处不连续、不可导，没法用梯度方法优化。

5.2 sigmoid 与"对数几率"

于是用一条光滑的 S 形曲线去近似阶跃函数，这就是 sigmoid（对数几率函数）：

$$\hat y=\sigma(z)=\frac{1}{1+e^{-z}},\qquad z=\mathbf{w}^\mathsf{T}\mathbf{x}+b$$

它把任意实数 $z$ 压到 $(0,1)$，正好可以解释成"样本属于类别 1 的概率"。$z\to+\infty$ 时趋近 1，$z\to-\infty$ 趋近 0，$z=0$ 时正好 0.5。预测规则：

$\hat y\ge 0.5 \Rightarrow$ 预测类别 1；$\quad\hat y<0.5\Rightarrow$ 预测类别 0。
（等价于看 $z\ge0$ 还是 $z<0$，因为 $\sigma(0)=0.5$。）

为什么叫"对数几率回归"：把式子反解一下会得到 $\ln\frac{\hat y}{1-\hat y}=\mathbf{w}^\mathsf{T}\mathbf{x}+b$。左边 $\frac{\hat y}{1-\hat y}$ 是"正类概率 ÷ 负类概率"，叫几率（odds），取对数叫对数几率（log-odds / logit）。所以这个模型本质是用线性回归去拟合对数几率——名字就是这么来的。这段了解即可，做题不用反解。

5.3 交叉熵损失

有了概率预测 $\hat y$，怎么衡量"预测得好不好"？用二分类交叉熵损失：

$$L=-\frac1N\sum_{i=1}^N\big[\,y_i\log\hat y_i+(1-y_i)\log(1-\hat y_i)\,\big]$$

直觉：当真实 $y_i=1$ 时，只剩 $-\log\hat y_i$，希望 $\hat y_i$ 越接近 1 损失越小；当 $y_i=0$ 时只剩 $-\log(1-\hat y_i)$，希望 $\hat y_i$ 越接近 0。预测对了损失趋于 0，错得离谱损失趋于无穷大。我们要做的就是调 $\mathbf{w},b$ 把 $L$ 降到最小。

5.4 梯度下降

$L$ 没有闭式解，但它是光滑凸函数，可以用梯度下降迭代逼近最小值。梯度下降的思想：从初值出发，每一步朝"损失下降最快"的反方向挪一小步，反复迭代。

交叉熵 + sigmoid 有一个非常漂亮的结论——梯度形式特别简洁（推导略，记住即可）：

$$w_j \leftarrow w_j-\alpha\cdot\frac1N\sum_{i=1}^N(\hat y_i-y_i)\,x_{ij}$$ $$b \leftarrow b-\alpha\cdot\frac1N\sum_{i=1}^N(\hat y_i-y_i)$$

其中 $\alpha$ 是学习率（步长），$(\hat y_i-y_i)$ 是预测概率与真实标签的误差。本题用批量梯度下降：每一轮用全部训练样本算一次平均梯度，更新一次参数，共做 $E$ 轮。

参数初始化（题目硬性规定）： $\mathbf{w}$ 全 0，$b=0$。这样保证大家训练轨迹一致、输出可复现。

数值稳定的 sigmoid

直接写 1/(1+exp(-z))，当 $z$ 是很大的负数时，exp(-z) 会溢出（overflow）报错。稳健写法是分情况：

import math
def sigmoid(z):
    if z >= 0:
        return 1.0 / (1.0 + math.exp(-z))
    else:
        ez = math.exp(z)              # z<0 时 exp(z) 不会溢出
        return ez / (1.0 + ez)

输入首行 N D Q E alpha；$N$ 行训练样本（$D$ 实数 + 标签 0/1）；$Q$ 行测试样本。输出$Q$ 行预测类别。

题目已给了完整骨架，你只要填三处 TODO。对照 5.4 的公式：

# 训练：每一轮(共 E 轮)用全部样本累加梯度，再平均、更新
for _ in range(E):
    grad_w = [0.0]*D
    grad_b = 0.0
    for i in range(N):
        # ① 线性输出 z = w·x + b
        z = sum(w[j]*train_X[i][j] for j in range(D)) + b
        # ② 预测概率
        y_hat = sigmoid(z)
        # ③ 误差 = 预测 - 真实
        error = y_hat - train_y[i]
        # 累加梯度
        for j in range(D):
            grad_w[j] += error * train_X[i][j]
        grad_b += error
    # 平均后更新(题目给的骨架已写好这部分)
    for j in range(D):
        w[j] -= alpha * grad_w[j] / N
    b -= alpha * grad_b / N

# 预测：概率 >= 0.5 输出 1，否则 0
for x in test_X:
    z = sum(w[j]*x[j] for j in range(D)) + b
    pred = 1 if sigmoid(z) >= 0.5 else 0
    predictions.append(pred)
题三易错点：
  1. 误差方向是 $\hat y - y$（预测减真实），不是反过来。梯度公式里正是这个符号让"更新 = 减去梯度"成立。
  2. 批量梯度：必须先把所有样本的梯度累加完，除以 $N$，再更新一次 $w,b$；不要每个样本就更新一次（那是随机梯度下降，结果对不上）。
  3. sigmoid 数值溢出：用上面的分段写法。
  4. $w,b$ 初始化全 0，别随机。
