第三部分 A · 运算方法与运算部件

核心思想:一切运算最终都归结为"加法器 + 移位器 + 多路选择器 + 控制逻辑"。先把加法器做快,再用它拼出乘除和浮点。

A.1 加法器的进化:从行波到先行进位(CLA)

上册 2.5+ 已经搭好了半加器 → 全加器 → 行波进位加法器,并算出行波加法器的延迟 ∝ n(2n 级门)这条瓶颈。这一节专门讲怎么打破它。延迟约定不变:与/或门 1ty,异或门 3ty。

回顾痛点:行波加法器为什么慢

n 个全加器级联,高位的进位输入必须等低位进位算完,进位像波纹逐级传:C0→Cn 要 2n 级门延迟,最高位的和要 2n+1 级。位数越多越慢——根子在于进位是串行依赖的。

先行进位(CLA)的核心思想:把进位"提前并行算出"

既然慢在"逐级等进位",那就不等——直接用最原始的输入 Aᵢ、Bᵢ 和 C0,并行把每一位的进位一次性算出来。

第一步,给每一位定义两个辅助信号(只看本位输入,彼此独立、可同时算): - 进位生成 Gᵢ = Aᵢ·Bᵢ:本位两个数都是 1 → 自己一定产生进位,不管低位。 - 进位传播 Pᵢ = Aᵢ⊕Bᵢ(也可用 Aᵢ+Bᵢ):本位至少一个 1 → 会把低位送来的进位"传"上去。

第二步,写出进位递推式:Cᵢ = Gᵢ + Pᵢ·Cᵢ₋₁(本位进位 = 本位生成 + 把低位进位传上来)。

第三步,逐层代入展开,把对低位进位的依赖全部消掉,变成只依赖 G、P、C0 的并行式(以 4 位为例):

C1 = G1 + P1·C0
C2 = G2 + P2·G1 + P2·P1·C0
C3 = G3 + P3·G2 + P3·P2·G1 + P3·P2·P1·C0
C4 = G4 + P4·G3 + P4·P3·G2 + P4·P3·P2·G1 + P4·P3·P2·P1·C0

看 C4:它直接由 G、P、C0 算出,完全不必等 C3!所以 C1~C4 可以同时产生——串行进位链被彻底打断。实现这套逻辑的电路叫 CLU(先行进位部件 / Carry Lookahead Unit)。

CLA 加法器的结构与延迟

  • 结构:n 个全加器只负责算 Gᵢ、Pᵢ 和最终的和 Sᵢ;中间所有进位交给一个 CLU 并行算,再把进位送回各全加器算和。
  • 延迟(4 位 CLA):1ty(算 G/P) + 2ty(CLU 两级与或出进位) + 3ty(各位 Sᵢ=…⊕Cᵢ₋₁ 的异或) = 6ty,与位数无关!对比 4 位行波的 2×4+1 = 9ty,而位数越大差距越悬殊。

代价与分组(多级先行)

  • 代价:位数越多,Cᵢ 的展开式越宽(C4 已经是 5 项、最长项 5 输入与门)→ 门的扇入/复杂度爆炸。所以不可能做任意宽的单级 CLA。
  • 解决:分组(组内并行、组间先行)。把宽加法器切成若干小组,组内用 CLA 并行,组间再用一层先行进位串起来(两级/多级 CLA)。例:32 位 = 4 个 8 位 CLA 组,总延迟约 12ty(远小于行波的 65ty)。
  • 一句话总结:CLA 用面积和门复杂度换速度,把进位的串行依赖改成并行计算——这是"用硬件换时间"的典型思路。
加法器 进位方式 延迟(n 位) 代价
行波进位 串行逐级传 ∝ n(2n 级) 简单、慢
单级 CLA 全部并行 常数级(约 6ty) 门扇入随 n 爆炸
分组多级 CLA 组内并行+组间先行 折中(如 32 位 ~12ty) 工程上的平衡点

A.2 带标志加法器与 ALU

在加法器输出端挂上标志位,运算结果的状态就能被指令读取: - ZF(零):结果全 0。SF(符号):结果最高位。 - OF(溢出)= Cₙ ⊕ Cₙ₋₁:带符号数溢出判断(最高位进位与次高位进位不同)。 - CF(进位/借位)= Cout ⊕ Cin:无符号数溢出与多精度运算。 - ALU(算术逻辑单元):加法器 + 逻辑运算 + 移位,用 ALUop(k 位)选择执行哪种操作(最多 2ᵏ 种)。ALU 是数据通路的"计算中枢"。

A.3 定点数加减运算

  • 补码加减(主力):A − B = A + [−B]补,统一成加法。减法时把减数取反加一送进加法器即可。溢出判断:OF = Cₙ ⊕ Cₙ₋₁,或用双符号位(变形补码)——结果两符号位不同即溢出(01 正溢、10 负溢)。
  • 原码加减:符号位与数值位分离处理,先判符号再对数值位求和/求差(求差用补码),按最高位进位定结果符号。求差永不溢出。
  • 移码加减:移码的和/差 = 和/差的补码,运算后把结果符号位取反还原成移码。用于浮点阶码。

A.4 定点数乘法:计算方法与原理证明

这一节把所有乘法算法串成一条逻辑链:先用最朴素的"无符号乘法"建立"部分积右移"的核心机制 → 原码乘法只是给它套上"符号异或";→ 补码乘法(Booth)用一个重编码恒等式让符号位也能直接参与;→ 两位乘法是把两步合并提速。每个算法都给出为什么成立的证明,看一遍就能懂。

A.4.0 共同基础:无符号数乘法(部分积右移)

两个 n 位无符号数相乘,按定义:

X × Y = X × (yₙ₋₁·2ⁿ⁻¹ + … + y₁·2¹ + y₀·2⁰) = Σᵢ X·yᵢ·2ⁱ

朴素做法:对每个 yᵢ=1 把 X 左移 i 位再相加——但被乘数要左移、需要 2n 位加法器,浪费。

改进(核心机制):不左移被乘数,而是每加一项后把部分积右移一位。这样加法永远作用在部分积的高 n 位,n 位加法器就够。其递推式(从乘数最低位 y₀ 开始,P₀=0):

Pᵢ₊₁ = 2⁻¹ · (Pᵢ + X·yᵢ)　　i = 0, 1, …, n−1
  • yᵢ=1 → 加 X 再右移;yᵢ=0 → 只右移。
  • 为什么右移等价于"项的 2ⁱ 权":第 i 步加进来的 X 此后还要被右移 (n−1−i) 次,每右移一次就再乘一个 2⁻¹;循环结束时所有项的权重恰好对齐成 X·yᵢ·2ⁱ 的总和。最终 Pₙ = X×Y。

这套"判乘数位 → 加/不加 → 右移"是所有定点乘法的母版,下面三种算法都只是改"判什么、加什么"。

A.4.1 原码一位乘法(用于浮点尾数)

原码的符号位和数值位是分离的,所以: - 积的符号 = 两数符号位异或; - 积的数值 = |X| × |Y|,直接套 A.4.0 的无符号乘法。

例:[x]原 = 0.1110,[y]原 = 1.1101。 - 数值:1110 × 1101 = 1011 0110;符号:0 ⊕ 1 = 1。 - 结果 [x×y]原 = 1.1011 0110。

A.4.2 原码两位乘法(提速一倍)

每次看乘数两位、循环 n/2 次。两位的权决定加几倍 X:00→+0、01→+X、10→+2X、11→+3X,然后部分积右移两位(相当于 ÷4)。

麻烦在 11(要 +3X):3X 无法一步加。利用 3X = 4X − X,把它拆成:

11: Pᵢ₊₁ = 2⁻²(Pᵢ + 3X) = 2⁻²(Pᵢ − X) + X

即本次做 −X,并用一个 T 触发器记下"下一轮欠一个 +X";因为部分积右移两位 = ÷4,下一轮补的那个 +X 正好等价于本轮的 +4X。于是判断要同时看两位 + T(共三位):

yᵢ₋₁ yᵢ T(上轮借位) 操作 T 置
00 0 +0 0
00 1 +X 0
01 0 +X 0
01 1 +2X 0
10 0 +2X 0
10 1 −X 1
11 0 −X 1
11 1 +0 1
  • −X 用 +[−X]补 实现;部分积采用模 8 补码(三位符号位)+ 算术右移——因为右移两位时若只用两位符号(模 4),中间部分积的符号会算错(变负),必须多留一位符号位。

例:[X]原 = 0.111001,[Y]原 = 0.100111 → [X×Y]原 = 0.1000 1010 1111。(速度快一倍,但硬件代价更大。)

A.4.3 补码一位乘法:Booth 算法(必考,带证明)

为什么不能直接用无符号乘法? 因为 [X×Y]补 ≠ [X]补 × [Y]补(补码的符号位带负权,直接相乘会错)。Booth 算法的精髓是用一个重编码恒等式让符号位也能统一参与。

证明(重编码):设 [Y]补 = yₙ₋₁…y₁y₀,其真值由补码定义:

Y = −yₙ₋₁·2ⁿ⁻¹ + yₙ₋₂·2ⁿ⁻² + … + y₀·2⁰

补一个 y₋₁ = 0,然后对每个权用恒等式 2ⁱ = 2ⁱ⁺¹ − 2ⁱ 改写、错位相消,可得Booth 重编码:

Y = Σᵢ (yᵢ₋₁ − yᵢ)·2ⁱ　　i = 0 … n−1

验证思路:把右边展开 = Σ yᵢ₋₁·2ⁱ − Σ yᵢ·2ⁱ;前一项换元 j=i−1 后,与后一项逐位合并,最高位 −yₙ₋₁·2ⁿ⁻¹ 自动带上负号——结果正好还原成补码真值公式。等式两边恒等,所以这个改写不改变 Y 的值。

两边乘 X 即得乘积递推:

X×Y = Σᵢ (yᵢ₋₁ − yᵢ)·X·2ⁱ　　→　　Pᵢ = 2⁻¹·(Pᵢ₋₁ + (yᵢ₋₁ − yᵢ)·X)

系数 (yᵢ₋₁ − yᵢ) 只可能是 −1 / 0 / +1,所以每步只要 看相邻两位 (yᵢ, yᵢ₋₁):

yᵢ yᵢ₋₁ 系数 操作(对部分积)
0 0 0 +0,算术右移
0 1 +1 +X(1 串结尾),算术右移
1 0 −1 −X(1 串开头),算术右移
1 1 0 +0,算术右移

直觉(1 串原理):一串连续的 1 满足 2ᵏ + … + 2ʲ = 2ᵏ⁺¹ − 2ʲ。所以遇到一长串 1,不必逐位加,只在串的开头(进入 1)减 X、串的结尾(离开 1)加 X——把许多次加法压成两次。

例:[X]补 = 1101(−3),[Y]补 = 0110(6),[−X]补 = 0011。逐位看 (yᵢ, yᵢ₋₁)(补 y₋₁=0):y₀y₋₁=00→移;y₁y₀=11→移;y₂y₁=01→+X;y₃y₂=10→−X。算下来 [X×Y]补 = 1110 1110 = −18 ✓。

A.4.4 补码两位乘法(提速 · 去年大题)

把相邻两步一位 Booth 合并,就得到两位 Booth:每次看三位窗口 yᵢ₊₁ yᵢ yᵢ₋₁、处理乘数两位、循环 n/2 次。

证明(合并):连写两步一位 Booth 递推并代入,

Pᵢ₊₂ = 2⁻¹(Pᵢ₊₁ + (yᵢ − yᵢ₊₁)·X)
　　　= 2⁻¹( 2⁻¹(Pᵢ + (yᵢ₋₁ − yᵢ)·X) + (yᵢ − yᵢ₊₁)·X )
　　　= 2⁻²·( Pᵢ + (yᵢ₋₁ + yᵢ − 2·yᵢ₊₁)·X )

合并系数 (yᵢ₋₁ + yᵢ − 2·yᵢ₊₁) 取值只有 {−2, −1, 0, +1, +2}。

关键好处:补码两位乘天然只需 0、±X、±2X,永远不会出现 +3X(系数最多到 2)。±2X 只是把 [X]补/[−X]补 左移一位。(原码两位乘才会撞上 +3X、要拆 −X+4X,见 A.4.2,别混。)

操作表(看三位窗口 → 加 → 算术右移两位):

yᵢ₊₁ yᵢ yᵢ₋₁ 系数 操作
0 0 0 0 +0
0 0 1 +1 +[X]补
0 1 0 +1 +[X]补
0 1 1 +2 +2[X]补
1 0 0 −2 +2[−X]补
1 0 1 −1 +[−X]补
1 1 0 −1 +[−X]补
1 1 1 0 +0

规则要点:① 补附加位 y₋₁=0;② 部分积最高位前再加一位附加符号位(防 ±2X 相加溢出),移位用算术右移;③ n 取偶数,循环 n/2 次;④ 结束时乘积高位在 P、低位在 Y。

完整例题:[X]补 = 1101(−3),[Y]补 = 0110(+6)。备 [−X]补=0011、2[X]补=11010、2[−X]补=00110,n=4 → 2 次循环。

步骤 P(部分积) Y(乘数) y₋₁ 说明
初始 00000 0110 0 P₀=0,补 y₋₁=0
看 y₁y₀y₋₁=100 +00110 系数 −2 → +2[−X]补
求和 00110 0110 0
算术右移 2 位 00001 1001 1 得 [P₂]补
看 y₃y₂y₁=011 +11010 系数 +2 → +2[X]补
求和 11011 1001
算术右移 2 位 11110 1110 得 [P₄]补

拼接 P‖Y 取低 8 位:[X×Y]补 = 1110 1110 = −18 ✓。与一位 Booth 结果相同,循环次数减半。

A.4.5 关于"移码乘法"的澄清

定点乘法没有"移码乘法"这一独立算法。 移码的用途只有两个:浮点数的阶码表示、以及移码加减(移码的和/差 = 和/差的补码,算完把符号位取反还原)。

在浮点乘法里移码确实会出现,但不是"把两个移码相乘":阶码做的是相加——而且因为两个移码各含一个偏置(E=e+Bias),直接相加会多出一个 Bias,需要减去一个偏置修正(E_积 = E_X + E_Y − Bias);真正相乘的是尾数(用上面的原码/补码乘法)。所以一提"乘法",移码就退场——它的战场是加减和阶码,不参与乘。

A.5 定点数除法(思路)

商逐位求,每步上商 + 移位 + 加减: - 恢复余数法:试减,若余数为负则把减掉的加回来(恢复),上商 0。 - 不恢复余数(加减交替)法:余数为负时不恢复,下一步改做加法,省去恢复步骤,更快。

A.6 浮点运算 IEEE 754

浮点加减(会做,重点)—— 五步

设两数 A = M_A × 2^(E_A)、B = M_B × 2^(E_B): 1. 对阶:把小阶向大阶看齐(小阶尾数右移,移动位数 = 阶差)。为什么小向大?大阶尾数右移会丢高位有效数字,损失更大。 2. 尾数加减:对齐后直接加减尾数。 3. 规格化:结果调整成 1.xxx 形式——尾数 ≥2 就右移、阶码 +1(右规);尾数 <1 就左移、阶码 −1(左规)。 4. 舍入:超出尾数位宽的部分按规则舍入(就近舍入等)。 5. 判溢出:阶码上溢 → ∞/异常;下溢 → 0 或非规格化数。

例:A = 1.01₂ × 2¹,B = 1.10₂ × 2⁻¹。 对阶:B 向阶 1 看齐,右移 2 位 → B = 0.0110 × 2¹。 尾数加:1.0100 + 0.0110 = 1.1010 × 2¹,已规格化。 结果 = 11.01₂ = 3.25。验证:A=2.5,B=0.75,和=3.25 ✓。

浮点乘除(了解)

  • 乘法:阶码相加(再减一个偏置)、尾数相乘,再规格化。
  • 除法:阶码相减、尾数相除。
  • 特殊处理(了解):舍入(就近/截断)、异常(上溢/下溢/无效)、精度(隐含位带来 1 位额外精度)。

第三部分 B · 指令集架构 (ISA)

B.1 ISA 是什么:软硬件的契约

ISA 是软件和硬件之间的分界契约:编译器/OS 只面向 ISA 编程,硬件可自由实现 ISA。它规定:指令集合与格式、操作数类型、寄存器组织、存储空间与编址、大小端、寻址方式、指令执行控制方式——但不规定电路怎么搭(那是微体系结构的事)。

B.2 操作数类型与存储方式

  • 操作数可来自:立即数(指令里直接给)、寄存器、内存。
  • 内存里多字节数据按大小端存放、按边界对齐(回顾上册 1.7)。

B.3 寻址方式(求有效地址 EA)

寻址方式 有效地址 EA 特点
立即 操作数在指令里 最快,无需访存
直接 EA = 指令给的地址 一次访存
间接 EA = M[指令给的地址] 多一次访存,灵活
寄存器(直接) 操作数 = R[r] 不访存,最快

B.4 操作类型 + 操作码编码

  • 操作类型:算术逻辑运算 / 移位 / 数据传送 / 顺序控制(转移) / 系统控制 / I/O。
  • 操作码编码:
  • 定长操作码:所有指令操作码位数相同,译码简单,但浪费编码空间。
  • 扩展(变长)操作码:给高频指令短码、低频指令长码,提高编码效率(类似哈夫曼思想)。

B.5 指令设计风格

  • 操作数风格:
  • 累加器型:隐含一个累加器,acc ← acc + M[A]。
  • 堆栈型:操作数隐含在栈顶。
  • 通用寄存器型:操作数在通用寄存器里。
  • 装入/存储型(load-store):运算指令只能碰寄存器,只有 load/store 访存——RISC 的基础。
  • 复杂度:RISC vs CISC
特性 CISC RISC
指令 多、变长、寻址方式多 少、定长、寻址方式少
访存 指令内可直接访存 仅 load/store 访存
指令周期 多数多周期 多数单周期(易流水)
代表 x86(VAX) RISC-V、ARM、MIPS

B.6 异常与中断(了解)

  • 异常:来自 CPU 内部(除零、地址越界、非法指令)。
  • 中断:来自 CPU 外部(I/O 设备、定时器)。
  • 共同点:都打断正常指令流,保存现场 → 转去处理程序 → 恢复现场。每个执行步骤都要检测。

B.7 RISC-V 指令系统(重点)

设计目标:规整、简洁、易流水。所有指令定长 32 位、按字地址对齐、采用 load-store 风格。

六种指令格式(必考:综合题 20 分)

格式 字段(高位→低位) 用途
R 型 funct7 \| rs2 \| rs1 \| funct3 \| rd \| opcode 寄存器-寄存器运算(add/slt)
I 型 imm[11:0] \| rs1 \| funct3 \| rd \| opcode 短立即数 / load(ori/lw)
S 型 imm[11:5] \| rs2 \| rs1 \| funct3 \| imm[4:0] \| opcode store(sw)
B 型 imm[12\|10:5] \| rs2 \| rs1 \| funct3 \| imm[4:1\|11] \| opcode 条件跳转(beq)
U 型 imm[31:12] \| rd \| opcode 长立即数(lui)
J 型 imm[20\|10:1\|11\|19:12] \| rd \| opcode 无条件跳转(jal)

立即数拼接(高频易错)

  • immI = 符号扩展 Instr[31:20](12 位)。
  • immU = {Instr[31:12], 12'b0}(低 12 位补 0,不是符号扩展)。
  • immS = 符号扩展 {Instr[31:25], Instr[11:7]}。
  • immB = {符号, Instr[7], Instr[30:25], Instr[11:8], 1'b0} —— 最低位强制为 0(指令按 2 字节对齐)。
  • immJ = {符号, Instr[19:12], Instr[20], Instr[30:21], 1'b0} —— 同样最低位补 0。
  • 记忆要点:B、J 型立即数位被打散重排,且最低位恒 0;U 型不符号扩展;其余符号扩展。

第三部分 C · CPU 设计(重点中的重点)

C.1 指令执行过程 + CPU 组成

一条指令的执行(公共流程): 取指 IR←M[PC] → PC+4 → 译码 → 取操作数(读寄存器)→ 执行(ALU)→ 访存(load/store)→ 写回 → 更新 PC。

CPU 两大块——这就是上册 FSM 模型的兑现: - 数据通路(Datapath):执行部件,负责数据存储/处理/传送。由组合逻辑元件(ALU、MUX、加法器,无记忆)和状态元件(PC、寄存器堆、存储器,有记忆)交替连成 状态元件 → 组合逻辑 → 状态元件。 - 控制器(Control Unit):对指令译码,生成控制信号,指挥数据通路动作(它就是那个 FSM)。

C.2 性能定量分析(必考填空/简答)

CPU 时间 = 指令条数 × CPI × 时钟周期(三要素缺一不可)。

  • CPI(每条指令平均时钟数)= 总时钟数 / 指令数;混合程序 CPI = Σ(CPIᵢ × Fᵢ)(Fᵢ 为指令频率)。
  • MIPS = 指令数 /(执行时间 ×10⁶)= 主频 /(CPI ×10⁶)。
  • 三要素各受谁影响:指令条数←算法/编译器/ISA;CPI←ISA/微体系结构;时钟周期←工艺/微体系结构。

  • 易错:不能只看主频或只看 MIPS 比较性能,必须三要素一起看(指令数 × CPI × 周期)。

C.3 单周期 CPU 数据通路(45 分大题级,务必吃透)

一条指令在一个时钟周期内走完全程。数据通路五大块:

9 条指令各走的通路(去年大题就在这套里挑):

指令 类型 做什么 关键控制信号
add R R[rd]←R[rs1]+R[rs2] ALUBSrc=0,RegWr=1
slt/sltu R 有/无符号比较置位 ALUctr=slt/sltu
ori I R[rd]←R[rs1] \| immI ALUBSrc=1(选立即数)
lui U R[rd]←imm<<12 ALU 直通 B(srcB)
lw I R[rd]←M[R[rs1]+immI] MemtoReg=1
sw S M[R[rs1]+immS]←R[rs2] MemWr=1,RegWr=0
beq B 相等则 PC←PC+immB Branch=1,ALU 做减判 Zero
jal J R[rd]←PC+4;PC←PC+immJ Jump=1,ALUASrc 选 PC

关于"×2"的澄清(易错):immB/immJ 的拼接规则末尾已补了 1'b0(见 B.7),所以它本身就是字节偏移(已经 ×2),目标地址 = PC + immB,不要再额外 ×2。真题里写成 PC + SEXT(imm[20:1] ≪ 1),那里的 imm[20:1] 是不含末位 0 的字段,≪1 才补上那个 0——和这里的 immB(含 0)是同一个值,只是切分口径不同,二者不可叠加。

  • 下地址逻辑(组合):Jump=1 选跳转目标;Branch=1 且 Zero=1 选分支目标;否则 PC+4。
  • 控制器实现:汇总 9 条指令的所有控制信号成真值表,用 PLA(与阵列译指令类型 → 或阵列出控制信号)实现;ALUctr 由指令类型 + funct3 译码。
  • 性能:CPI = 1,但时钟周期由最长指令 lw 的关键路径决定(取指+读寄存器+ALU+访存+写回)。缺点:add 这种短指令也被迫陪绑长周期,浪费时间——这正是多周期/流水线要解决的。

C.4 多周期 CPU(了解)

  • 把指令拆成多个阶段,每阶段一个较短周期,不同指令用不同周期数;功能部件可跨周期复用(指令/数据存储器合一、ALU 复用)。
  • 典型周期数:Load 5、Store 4、R 型 4、Branch 3、Jump 3。
  • CPI = Σ(占比ᵢ × 周期数ᵢ)。例:0.22×5 + 0.11×4 + 0.49×4 + 0.16×3 + 0.02×3 = 4.04。周期约为单周期的 1/5,总时间约 0.8 倍。
  • 控制器两种:硬连线(FSM+PLA,快但难维护)、微程序(控制信号固化在控存 ROM,水平型并行度高/垂直型编码紧凑)。

C.5 流水线 CPU(了解)

  • 经典五级:IF(取指)→ ID(译码/读寄存器)→ EX(执行)→ MEM(访存)→ WB(写回)。所有指令对齐五段,缺段补 NOP。理想 CPI → 1 且周期短。
  • 三种冒险:
  • 结构冒险:部件争用 → 指令/数据 Cache 分离。
  • 数据冒险:前指令结果未写回就被用 → 转发(Forwarding) 解决;load-use 必须阻塞 1 拍 + 转发。
  • 控制冒险:转移目标未定就取错指令 → 分支预测(静态 65–85% / 动态 90%+)、延迟分支、清空错误指令。

全课闭环小结

  • 上册:0/1 表示信息(第一部分)→ 布尔代数 + 门电路 + 组合/时序逻辑造出会算的电路(第二部分)。
  • 下册:加法器拼出定点/浮点运算(A)→ ISA 定义软硬件契约(B)→ 数据通路 + 控制器把 ISA 实现成 CPU(C)。
  • 一条主线贯穿到底:信息 → 组件 → 计算机;而 CPU = 放大版 FSM(数据通路=组合,控制器=时序)正是第二部分那把钥匙在第三部分的兑现。
